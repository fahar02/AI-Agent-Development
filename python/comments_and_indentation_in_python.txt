1. WHAT ARE COMMENTS?
================================================================================

DEFINITION:

    Comments are special lines in your code that Python IGNORES when running 
    the program. They are notes written by programmers for humans to read, 
    not for the computer to execute.

    Think of comments as sticky notes you put on your code to explain what 
    it does or why you wrote it that way.


REAL-LIFE ANALOGY:

    Imagine you're writing a recipe:
    
    Without Comments (Just Instructions):
        Add 2 cups flour
        Add 1 cup sugar
        Mix for 5 minutes
    
    With Comments (Helpful Notes):
        Add 2 cups flour (use all-purpose flour)
        Add 1 cup sugar (white sugar works best)
        Mix for 5 minutes (until smooth and creamy)
    
    The notes in parentheses are like comments - they help you understand 
    WHY and HOW, but they're not part of the actual cooking steps.


WHY COMMENTS ARE IMPORTANT:

1. EXPLAIN YOUR CODE:
   Comments help others (and your future self) understand what your code does.

2. MAKE CODE READABLE:
   Good comments make complex code easier to understand.

3. DOCUMENT DECISIONS:
   Explain WHY you chose a particular approach.

4. DEBUGGING HELP:
   You can temporarily disable code by commenting it out.

5. COLLABORATION:
   Help team members understand your code quickly.

6. LEARNING TOOL:
   Great for beginners to add notes while learning.


EXAMPLE: Code Without vs With Comments

    Without Comments:
    
        x = 10
        y = 20
        z = x + y
        print(z)
    
    
    With Comments:
    
        x = 10
        y = 20
        z = x + y
        print(z)
    
    
    With HELPFUL Comments:
    
        price_per_item = 10
        quantity = 20
        total_price = price_per_item * quantity
        print(total_price)

2. TYPES OF COMMENTS IN PYTHON
================================================================================

Python has TWO main ways to write comments:

1. Single-line comments (using #)
2. Multi-line comments (using triple quotes)


--------------------------------------------------------------------------------
TYPE 1: SINGLE-LINE COMMENTS
--------------------------------------------------------------------------------

SYNTAX:
    # This is a comment

    Anything after the # symbol on that line is ignored by Python.


EXAMPLE 1: Comment on Its Own Line

    # This program calculates the area of a rectangle
    length = 10
    width = 5
    area = length * width
    print(area)


EXAMPLE 2: Comment at the End of a Line (Inline Comment)

    length = 10
    width = 5
    area = length * width
    print(area)


EXAMPLE 3: Multiple Single-Line Comments

    age = 25
    
    
    if age >= 18:
        print("You are an adult")


EXAMPLE 4: Commenting Out Code (Temporarily Disable)

    name = "John"
    age = 25
    
    print(name)
    print(age)


--------------------------------------------------------------------------------
TYPE 2: MULTI-LINE COMMENTS
--------------------------------------------------------------------------------

SYNTAX:
    """
    This is a multi-line comment.
    It can span multiple lines.
    """
    
    OR
    
    '''
    This is also a multi-line comment.
    Using single quotes instead of double quotes.
    '''


IMPORTANT NOTE:
    Technically, these are called "docstrings" (documentation strings), not 
    true comments. Python doesn't completely ignore them - they can be accessed 
    as documentation. But for beginners, you can think of them as multi-line 
    comments.


EXAMPLE 1: Multi-Line Comment at the Top

    """
    Program: Simple Calculator
    Author: Student Name
    Date: April 1, 2026
    Purpose: This program adds two numbers entered by the user
    """
    
    num1 = int(input("Enter first number: "))
    num2 = int(input("Enter second number: "))
    result = num1 + num2
    print("Sum:", result)


EXAMPLE 2: Explaining Complex Logic

    def calculate_discount(price, customer_type):
        '''
        This function calculates discount based on customer type.
        Regular customers get 5% discount.
        Premium customers get 15% discount.
        VIP customers get 25% discount.
        '''
        
        if customer_type == "regular":
            discount = price * 0.05
        elif customer_type == "premium":
            discount = price * 0.15
        elif customer_type == "vip":
            discount = price * 0.25
        else:
            discount = 0
        
        return discount


EXAMPLE 3: Section Dividers

    """
    ========================================
    SECTION 1: INPUT FROM USER
    ========================================
    """
    name = input("Enter name: ")
    age = int(input("Enter age: "))
    
    """
    ========================================
    SECTION 2: PROCESS DATA
    ========================================
    """
    birth_year = 2026 - age
    
    """
    ========================================
    SECTION 3: DISPLAY OUTPUT
    ========================================
    """
    print(f"{name} was born in {birth_year}")

3. BEST PRACTICES FOR WRITING COMMENTS
================================================================================

DO'S:

1. WRITE CLEAR AND SIMPLE COMMENTS:
   
   Good:
       radius = 5
   
   Bad:
       r = 5


2. EXPLAIN WHY, NOT WHAT (when obvious):
   
   Bad:
       x = x + 1
   
   Good:
       x = x + 1


3. UPDATE COMMENTS WHEN CODE CHANGES:
   
   If you change code, update the comment too!


4. USE COMMENTS FOR COMPLEX LOGIC:
   
   total_with_tax = price * 1.18


5. ADD COMMENTS FOR FUTURE REFERENCE:
   
   


DON'TS:

1. DON'T STATE THE OBVIOUS:
   
   Bad:
       x = 5


2. DON'T WRITE MISLEADING COMMENTS:
   
   Wrong Comment:
       x = x + 1


3. DON'T OVER-COMMENT:
   
   Too Much:
       name = "John"
       age = 25
       city = "Delhi"


4. DON'T USE COMMENTS TO EXPLAIN BAD CODE:
   
   Bad:
       
       x = a * b + c / d - e
   
   Better:
       base_amount = price * quantity
       tax = base_amount * tax_rate
       discount = base_amount * discount_rate
       total = base_amount + tax - discount


WHEN TO USE COMMENTS:

Use comments when:
- Explaining complex algorithms
- Documenting function purposes
- Noting important decisions
- Warning about potential issues
- Providing examples of usage
- Marking TODO items

Don't use comments when:
- The code is self-explanatory
- Variable names are clear
- Logic is simple and obvious

4. WHAT IS INDENTATION?
================================================================================

DEFINITION:

    Indentation means the spaces or tabs at the beginning of a line of code.
    In Python, indentation is NOT just for making code look pretty - it's 
    actually part of the syntax!

    Indentation tells Python which lines of code belong together in a block.


REAL-LIFE ANALOGY:

    Think of indentation like organizing items in a drawer:
    
    Main Drawer (No Indentation)
        Small Box 1 (Indented - inside main drawer)
            Item A (More Indented - inside small box 1)
            Item B (More Indented - inside small box 1)
        Small Box 2 (Indented - inside main drawer)
            Item C (More Indented - inside small box 2)
    
    The indentation shows what's inside what!
    
    Similarly in Python:
    
    if condition:
        statement 1
        statement 2
        if another_condition:
            statement 3
            statement 4
        statement 5


WHY INDENTATION IS IMPORTANT IN PYTHON:

1. PYTHON REQUIRES IT:
   Unlike many other languages, Python uses indentation to define code blocks.
   Other languages use curly braces {}, but Python uses indentation.

2. MAKES CODE READABLE:
   Proper indentation makes code easy to read and understand.

3. SHOWS CODE STRUCTURE:
   Indentation clearly shows which code belongs to which block.

4. PREVENTS ERRORS:
   Wrong indentation causes errors in Python!


COMPARISON WITH OTHER LANGUAGES:

    JavaScript (uses curly braces):
    
        if (age >= 18) {
            console.log("Adult");
            console.log("Can vote");
        }
    
    
    Python (uses indentation):
    
        if age >= 18:
            print("Adult")
            print("Can vote")
    
    
    Notice: Python doesn't need curly braces because indentation does the job!


================================================================================
5. ROLE OF INDENTATION IN PYTHON
================================================================================

INDENTATION DEFINES CODE BLOCKS:

    A code block is a group of statements that belong together.
    In Python, we use indentation to create code blocks.


--------------------------------------------------------------------------------
INDENTATION IN IF STATEMENTS:
--------------------------------------------------------------------------------

EXAMPLE 1: Simple If Statement

    age = 20
    
    if age >= 18:
        print("You are an adult")
        print("You can vote")
    
    print("This line runs regardless")

    Output:
        You are an adult
        You can vote
        This line runs regardless
    
    Explanation:
    - Lines with indentation run only if condition is True
    - Line without indentation runs always


EXAMPLE 2: If-Else Statement

    age = 15
    
    if age >= 18:
        print("You are an adult")
        print("You can vote")
    else:
        print("You are a minor")
        print("You cannot vote yet")
    
    print("Thank you!")

    Output:
        You are a minor
        You cannot vote yet
        Thank you!


EXAMPLE 3: Nested If Statements

    age = 25
    has_license = True
    
    if age >= 18:
        print("You are old enough to drive")
        
        if has_license:
            print("You can drive legally")
            print("Drive safely!")
        else:
            print("Please get a license first")
    else:
        print("You are too young to drive")

    Output:
        You are old enough to drive
        You can drive legally
        Drive safely!


--------------------------------------------------------------------------------
INDENTATION IN LOOPS:
--------------------------------------------------------------------------------

EXAMPLE 1: For Loop

    for i in range(5):
        print(f"Number: {i}")
        print("Inside loop")
    
    print("Outside loop")

    Output:
        Number: 0
        Inside loop
        Number: 1
        Inside loop
        Number: 2
        Inside loop
        Number: 3
        Inside loop
        Number: 4
        Inside loop
        Outside loop


EXAMPLE 2: While Loop

    count = 1
    
    while count <= 3:
        print(f"Count is {count}")
        count = count + 1
    
    print("Loop finished")

    Output:
        Count is 1
        Count is 2
        Count is 3
        Loop finished


EXAMPLE 3: Nested Loops

    for i in range(1, 4):
        print(f"Outer loop: {i}")
        
        for j in range(1, 3):
            print(f"  Inner loop: {j}")
        
        print("Back to outer loop")

    Output:
        Outer loop: 1
          Inner loop: 1
          Inner loop: 2
        Back to outer loop
        Outer loop: 2
          Inner loop: 1
          Inner loop: 2
        Back to outer loop
        Outer loop: 3
          Inner loop: 1
          Inner loop: 2
        Back to outer loop


--------------------------------------------------------------------------------
INDENTATION IN FUNCTIONS:
--------------------------------------------------------------------------------

EXAMPLE 1: Simple Function

    def greet():
        print("Hello!")
        print("Welcome to Python")
    
    greet()

    Output:
        Hello!
        Welcome to Python


EXAMPLE 2: Function with Parameters

    def add_numbers(a, b):
        result = a + b
        print(f"Sum: {result}")
        return result
    
    answer = add_numbers(5, 3)

    Output:
        Sum: 8


EXAMPLE 3: Function with Conditions

    def check_grade(marks):
        print(f"Marks obtained: {marks}")
        
        if marks >= 90:
            print("Grade: A+")
        elif marks >= 80:
            print("Grade: A")
        elif marks >= 70:
            print("Grade: B")
        else:
            print("Grade: C")
    
    check_grade(85)

    Output:
        Marks obtained: 85
        Grade: A

6. INDENTATION ERRORS AND EXAMPLES
================================================================================

CORRECT INDENTATION:

    age = 20
    
    if age >= 18:
        print("Adult")
        print("Can vote")

    This works perfectly!


--------------------------------------------------------------------------------
ERROR 1: IndentationError - Expected an Indented Block
--------------------------------------------------------------------------------

WRONG CODE:

    age = 20
    
    if age >= 18:
    print("Adult")

ERROR MESSAGE:
    IndentationError: expected an indented block


EXPLANATION:
    After the colon (:), Python expects an indented block.
    The print statement should be indented.


CORRECT CODE:

    age = 20
    
    if age >= 18:
        print("Adult")


--------------------------------------------------------------------------------
ERROR 2: IndentationError - Unexpected Indent
--------------------------------------------------------------------------------

WRONG CODE:

    age = 20
        print("Hello")

ERROR MESSAGE:
    IndentationError: unexpected indent


EXPLANATION:
    The print statement is indented but there's no reason for it.
    It's not inside any block (if, loop, function, etc.).


CORRECT CODE:

    age = 20
    print("Hello")


--------------------------------------------------------------------------------
ERROR 3: IndentationError - Unindent Does Not Match
--------------------------------------------------------------------------------

WRONG CODE:

    if age >= 18:
        print("Adult")
          print("Can vote")

ERROR MESSAGE:
    IndentationError: unindent does not match any outer indentation level


EXPLANATION:
    The second print has different indentation (more spaces) than the first.
    All statements in the same block must have the same indentation.


CORRECT CODE:

    if age >= 18:
        print("Adult")
        print("Can vote")


--------------------------------------------------------------------------------
ERROR 4: Mixing Tabs and Spaces
--------------------------------------------------------------------------------

WRONG CODE:

    if age >= 18:
        print("Adult")
    	print("Can vote")

ERROR MESSAGE:
    TabError: inconsistent use of tabs and spaces in indentation


EXPLANATION:
    First line uses spaces, second line uses tab.
    Never mix tabs and spaces!


CORRECT CODE:

    if age >= 18:
        print("Adult")
        print("Can vote")


--------------------------------------------------------------------------------
ERROR 5: Wrong Logic Due to Indentation
--------------------------------------------------------------------------------

WRONG CODE (No Error, But Wrong Logic):

    age = 15
    
    if age >= 18:
        print("You are an adult")
    print("You can vote")

    Output:
        You can vote


EXPLANATION:
    The second print runs even when age < 18 because it's not indented.
    This is not a syntax error, but a logic error!


CORRECT CODE:

    age = 15
    
    if age >= 18:
        print("You are an adult")
        print("You can vote")

    Output:
        (nothing printed)

7. INDENTATION RULES AND STANDARDS
================================================================================

RULE 1: USE CONSISTENT INDENTATION

    Choose either spaces OR tabs, and stick with it throughout your code.
    
    Recommended: Use SPACES (not tabs)


RULE 2: STANDARD INDENTATION IS 4 SPACES

    According to PEP 8 (Python's official style guide), use 4 spaces for 
    each indentation level.
    
    Good:
        if condition:
            statement
    
    Acceptable but not recommended:
        if condition:
          statement
    
    Bad:
        if condition:
         statement


RULE 3: ALL STATEMENTS IN A BLOCK MUST HAVE SAME INDENTATION

    Good:
        if age >= 18:
            print("Line 1")
            print("Line 2")
            print("Line 3")
    
    Bad:
        if age >= 18:
            print("Line 1")
              print("Line 2")
            print("Line 3")


RULE 4: NESTED BLOCKS NEED ADDITIONAL INDENTATION

    if condition1:
        statement1
        
        if condition2:
            statement2
            statement3
        
        statement4


RULE 5: BLANK LINES DON'T NEED INDENTATION

    if age >= 18:
        print("Adult")
        
        print("Can vote")


PEP 8 BASICS (Python Enhancement Proposal 8):

    PEP 8 is the official style guide for Python code.
    
    Key Points:
    
    1. Use 4 spaces per indentation level
    2. Never mix tabs and spaces
    3. Limit lines to 79 characters (for readability)
    4. Use blank lines to separate functions and classes
    5. Use spaces around operators: x = 5 (not x=5)
    6. Use lowercase with underscores for variable names: my_variable


HOW TO SET UP YOUR EDITOR:

    Most code editors can be configured to:
    
    1. Convert tabs to spaces automatically
    2. Show indentation guides (vertical lines)
    3. Highlight indentation errors
    4. Auto-indent after colons
    
    Popular editors: VS Code, PyCharm, IDLE, Sublime Text

9. ASCII DIAGRAM: INDENTATION STRUCTURE
================================================================================

CORRECT INDENTATION:

    if condition:
        statement 1
        statement 2
        
        if nested_condition:
            statement 3
            statement 4
        
        statement 5
    
    statement 6


VISUAL REPRESENTATION:

    Level 0 (No Indent)
    |
    +-- Level 1 (4 spaces)
    |   |
    |   +-- Level 2 (8 spaces)
    |   |
    |   +-- Back to Level 1
    |
    +-- Back to Level 0


FLOW DIAGRAM:

    START
      |
      v
    [if condition:]  <--- No indentation
      |
      +---> [statement 1]  <--- 4 spaces indentation
      |
      +---> [statement 2]  <--- 4 spaces indentation
      |
      +---> [if nested_condition:]  <--- 4 spaces indentation
      |       |
      |       +---> [statement 3]  <--- 8 spaces indentation
      |       |
      |       +---> [statement 4]  <--- 8 spaces indentation
      |
      +---> [statement 5]  <--- 4 spaces indentation
      |
      v
    [statement 6]  <--- No indentation (outside if block)
      |
      v
    END


ERROR VISUALIZATION:

    WRONG (No Indentation):
    
    if condition:
    statement 1  <--- ERROR! Expected indent
    
    
    WRONG (Inconsistent Indentation):
    
    if condition:
        statement 1  <--- 4 spaces
          statement 2  <--- 6 spaces ERROR!
    
    
    CORRECT:
    
    if condition:
        statement 1  <--- 4 spaces
        statement 2  <--- 4 spaces

10. COMMON BEGINNER MISTAKES
================================================================================

MISTAKE 1: Forgetting to Indent After Colon

    Wrong:
        if age >= 18:
        print("Adult")
    
    Correct:
        if age >= 18:
            print("Adult")


MISTAKE 2: Over-Indenting

    Wrong:
        age = 20
            print(age)
    
    Correct:
        age = 20
        print(age)


MISTAKE 3: Inconsistent Indentation in Same Block

    Wrong:
        if True:
            print("Line 1")
              print("Line 2")
            print("Line 3")
    
    Correct:
        if True:
            print("Line 1")
            print("Line 2")
            print("Line 3")


MISTAKE 4: Not Indenting All Statements in Block

    Wrong:
        if age >= 18:
            print("Adult")
        print("Can vote")
        print("Has rights")
    
    If you want all three to be in the if block:
        if age >= 18:
            print("Adult")
            print("Can vote")
            print("Has rights")


MISTAKE 5: Mixing Tabs and Spaces

    Wrong:
        if True:
            print("Spaces")
        	print("Tab")
    
    Correct:
        if True:
            print("Spaces")
            print("Spaces")


MISTAKE 6: Commenting Without Proper Indentation

    Wrong:
        if age >= 18:
        print("Adult")
    
    Correct:
        if age >= 18:
            print("Adult")

11. TIPS FOR WRITING CLEAN PYTHON CODE
================================================================================

TIP 1: Use 4 Spaces for Indentation

    Always use 4 spaces. Configure your editor to convert tabs to 4 spaces.


TIP 2: Be Consistent

    If you start with spaces, use spaces throughout.
    Don't switch between tabs and spaces.


TIP 3: Use Comments Wisely

    Comment complex logic, not obvious code.
    Keep comments up-to-date with code changes.


TIP 4: Add Blank Lines for Readability

    Use blank lines to separate logical sections:
    
    name = input("Enter name: ")
    age = int(input("Enter age: "))
    
    if age >= 18:
        print(f"{name} is an adult")
    else:
        print(f"{name} is a minor")


TIP 5: Use Meaningful Names Instead of Comments

    Bad:
        x = 10
    
    Good:
        student_age = 10


TIP 6: Follow PEP 8 Guidelines

    Read PEP 8 style guide for Python best practices.


TIP 7: Use a Good Code Editor

    Use editors like VS Code, PyCharm that show indentation guides.


TIP 8: Check Your Code Visually

    Look at your code - does it look organized?
    Proper indentation makes code look clean.


TIP 9: Test Your Code

    Run your code frequently to catch indentation errors early.


TIP 10: Learn from Examples

    Read well-written Python code to see good practices.
