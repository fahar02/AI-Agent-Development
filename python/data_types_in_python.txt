1. WHAT ARE DATA TYPES?
================================================================================

A data type is like a LABEL that tells Python what kind of information you are
storing in a variable.

REAL-LIFE ANALOGY:
Think of data types like containers in your kitchen:
- A water bottle holds liquids (like float numbers)
- A jar holds solid items (like integers)
- A basket holds multiple items (like lists)

Each container is designed for a specific purpose. Similarly, each data type
in Python is designed to store a specific kind of information.


================================================================================
2. WHY DATA TYPES MATTER
================================================================================

Data types are important because:

1. MEMORY EFFICIENCY: Different types use different amounts of memory
2. OPERATIONS: You can do different things with different types
3. ERROR PREVENTION: Helps avoid mistakes in your code
4. PERFORMANCE: Proper types make your program faster

Example of why it matters:

    age = 25              # This is a number (int)
    name = "25"           # This is text (str)
    
    print(age + 5)        # Output: 30 (math works!)
    print(name + 5)       # ERROR! Cannot add text and number


================================================================================
3. BUILT-IN DATA TYPES IN PYTHON
================================================================================

Python has several built-in data types organized into categories:

NUMERIC TYPES:
    - int       (whole numbers)
    - float     (decimal numbers)
    - complex   (complex numbers with real and imaginary parts)

TEXT TYPE:
    - str       (string - text data)

SEQUENCE TYPES:
    - list      (ordered, changeable collection)
    - tuple     (ordered, unchangeable collection)
    - range     (sequence of numbers)

MAPPING TYPE:
    - dict      (key-value pairs)

SET TYPES:
    - set       (unordered, unique items)
    - frozenset (unchangeable set)

BOOLEAN TYPE:
    - bool      (True or False)

NONE TYPE:
    - NoneType  (represents absence of value)


================================================================================
4. NUMERIC TYPES (int, float, complex)
================================================================================

--------------------------------------------------------------------------------
4.1 INTEGER (int)
--------------------------------------------------------------------------------

WHAT: Whole numbers without decimal points (positive, negative, or zero)

WHY: Used for counting, indexing, and whole number calculations

HOW: Simply write the number without quotes

Examples:

    # Positive integers
    age = 25
    students = 100
    year = 2024
    
    # Negative integers
    temperature = -15
    debt = -5000
    
    # Zero
    balance = 0
    
    # Large numbers (Python handles any size!)
    population = 8000000000
    big_number = 999999999999999999999999999999
    
    print(age)              # Output: 25
    print(type(age))        # Output: <class 'int'>


REAL-WORLD USE CASES:
    - Counting items in a shopping cart
    - Tracking page numbers in a book
    - Storing age, year, quantity


--------------------------------------------------------------------------------
4.2 FLOAT (float)
--------------------------------------------------------------------------------

WHAT: Numbers with decimal points

WHY: Used for precise measurements and calculations

HOW: Write numbers with a decimal point

Examples:

    # Decimal numbers
    price = 19.99
    temperature = 36.6
    pi = 3.14159
    
    # Scientific notation
    speed_of_light = 3e8        # 3 x 10^8 = 300000000.0
    small_number = 1.5e-3       # 1.5 x 10^-3 = 0.0015
    
    # Division always gives float
    result = 10 / 4
    print(result)               # Output: 2.5
    print(type(result))         # Output: <class 'float'>
    
    # Even when dividing evenly
    result2 = 10 / 2
    print(result2)              # Output: 5.0 (still a float!)


REAL-WORLD USE CASES:
    - Calculating prices with cents
    - Measuring height, weight
    - Scientific calculations


--------------------------------------------------------------------------------
4.3 COMPLEX (complex)
--------------------------------------------------------------------------------

WHAT: Numbers with real and imaginary parts (uses 'j' for imaginary unit)

WHY: Used in advanced mathematics, engineering, and physics

HOW: Write as: real + imaginary_j

Examples:

    # Complex numbers
    num1 = 3 + 4j
    num2 = 2 - 5j
    num3 = 1j           # Only imaginary part
    
    print(num1)                 # Output: (3+4j)
    print(type(num1))           # Output: <class 'complex'>
    
    # Accessing parts
    print(num1.real)            # Output: 3.0
    print(num1.imag)            # Output: 4.0
    
    # Operations
    result = num1 + num2
    print(result)               # Output: (5-1j)


REAL-WORLD USE CASES:
    - Electrical engineering calculations
    - Signal processing
    - Quantum physics

5. STRING TYPE (str)
================================================================================

WHAT: Text data - sequence of characters

WHY: Used to store and manipulate text

HOW: Enclose text in single quotes '', double quotes "", or triple quotes """

Examples:

    # Single quotes
    name = 'John'
    
    # Double quotes
    city = "New York"
    
    # Triple quotes (for multiple lines)
    message = """This is a
    multi-line
    string"""
    
    # Empty string
    empty = ""
    
    # String with numbers (still text!)
    phone = "1234567890"
    
    print(name)                 # Output: John
    print(type(name))           # Output: <class 'str'>


STRING OPERATIONS:

    # Concatenation (joining)
    first_name = "John"
    last_name = "Doe"
    full_name = first_name + " " + last_name
    print(full_name)            # Output: John Doe
    
    # Repetition
    laugh = "ha" * 3
    print(laugh)                # Output: hahaha
    
    # Indexing (starts from 0)
    word = "Python"
    print(word[0])              # Output: P
    print(word[3])              # Output: h
    print(word[-1])             # Output: n (last character)
    
    # Slicing
    print(word[0:3])            # Output: Pyt
    print(word[2:])             # Output: thon
    
    # Length
    print(len(word))            # Output: 6
    
    # Methods
    text = "hello world"
    print(text.upper())         # Output: HELLO WORLD
    print(text.capitalize())    # Output: Hello world
    print(text.replace("world", "Python"))  # Output: hello Python


ESCAPE CHARACTERS:

    # Newline
    print("Hello\nWorld")       # Output: Hello
                                #         World
    
    # Tab
    print("Name:\tJohn")        # Output: Name:    John
    
    # Quotes inside strings
    quote = "He said, \"Hello!\""
    print(quote)                # Output: He said, "Hello!"


REAL-WORLD USE CASES:
    - Storing names, addresses
    - User input/output
    - File paths, URLs

6. SEQUENCE TYPES (list, tuple)
================================================================================

--------------------------------------------------------------------------------
6.1 LIST
--------------------------------------------------------------------------------

WHAT: Ordered collection of items that can be changed (mutable)

WHY: Store multiple related items together

HOW: Use square brackets []

Examples:

    # Creating lists
    fruits = ["apple", "banana", "orange"]
    numbers = [1, 2, 3, 4, 5]
    mixed = [1, "hello", 3.14, True]
    empty_list = []
    
    print(fruits)               # Output: ['apple', 'banana', 'orange']
    print(type(fruits))         # Output: <class 'list'>


LIST OPERATIONS:

    # Accessing items (indexing)
    fruits = ["apple", "banana", "orange"]
    print(fruits[0])            # Output: apple
    print(fruits[-1])           # Output: orange
    
    # Modifying items
    fruits[1] = "mango"
    print(fruits)               # Output: ['apple', 'mango', 'orange']
    
    # Adding items
    fruits.append("grape")      # Add at end
    print(fruits)               # Output: ['apple', 'mango', 'orange', 'grape']
    
    fruits.insert(1, "kiwi")    # Insert at position 1
    print(fruits)               # Output: ['apple', 'kiwi', 'mango', 'orange', 'grape']
    
    # Removing items
    fruits.remove("mango")      # Remove by value
    print(fruits)               # Output: ['apple', 'kiwi', 'orange', 'grape']
    
    popped = fruits.pop()       # Remove last item
    print(popped)               # Output: grape
    print(fruits)               # Output: ['apple', 'kiwi', 'orange']
    
    # Length
    print(len(fruits))          # Output: 3
    
    # Checking membership
    print("apple" in fruits)    # Output: True
    print("mango" in fruits)    # Output: False
    
    # Slicing
    numbers = [0, 1, 2, 3, 4, 5]
    print(numbers[1:4])         # Output: [1, 2, 3]
    print(numbers[:3])          # Output: [0, 1, 2]
    print(numbers[3:])          # Output: [3, 4, 5]
    
    # Sorting
    nums = [3, 1, 4, 1, 5, 9, 2]
    nums.sort()
    print(nums)                 # Output: [1, 1, 2, 3, 4, 5, 9]
    
    # Reversing
    nums.reverse()
    print(nums)                 # Output: [9, 5, 4, 3, 2, 1, 1]


NESTED LISTS:

    # List inside list
    matrix = [
        [1, 2, 3],
        [4, 5, 6],
        [7, 8, 9]
    ]
    
    print(matrix[0])            # Output: [1, 2, 3]
    print(matrix[1][2])         # Output: 6


REAL-WORLD USE CASES:
    - Shopping cart items
    - Student grades
    - To-do list tasks


--------------------------------------------------------------------------------
6.2 TUPLE
--------------------------------------------------------------------------------

WHAT: Ordered collection of items that CANNOT be changed (immutable)

WHY: Store data that should not be modified

HOW: Use parentheses ()

Examples:

    # Creating tuples
    coordinates = (10, 20)
    colors = ("red", "green", "blue")
    single_item = (42,)         # Note the comma for single item!
    mixed = (1, "hello", 3.14)
    empty_tuple = ()
    
    print(coordinates)          # Output: (10, 20)
    print(type(coordinates))    # Output: <class 'tuple'>


TUPLE OPERATIONS:

    # Accessing items
    point = (10, 20, 30)
    print(point[0])             # Output: 10
    print(point[-1])            # Output: 30
    
    # Cannot modify (this will cause ERROR!)
    # point[0] = 15             # TypeError: 'tuple' object does not support item assignment
    
    # Length
    print(len(point))           # Output: 3
    
    # Checking membership
    print(20 in point)          # Output: True
    
    # Unpacking
    x, y, z = point
    print(x)                    # Output: 10
    print(y)                    # Output: 20
    print(z)                    # Output: 30
    
    # Concatenation (creates new tuple)
    tuple1 = (1, 2)
    tuple2 = (3, 4)
    result = tuple1 + tuple2
    print(result)               # Output: (1, 2, 3, 4)
    
    # Repetition
    repeated = (1, 2) * 3
    print(repeated)             # Output: (1, 2, 1, 2, 1, 2)


WHEN TO USE TUPLE VS LIST:

    Use TUPLE when:
    - Data should not change (dates, coordinates)
    - You want to protect data from modification
    - Slightly faster than lists
    
    Use LIST when:
    - Data needs to be modified
    - Adding/removing items frequently


REAL-WORLD USE CASES:
    - GPS coordinates (latitude, longitude)
    - RGB color values
    - Database records

7. SET TYPE (set)
================================================================================

WHAT: Unordered collection of UNIQUE items

WHY: Automatically removes duplicates, fast membership testing

HOW: Use curly braces {} or set()

Examples:

    # Creating sets
    fruits = {"apple", "banana", "orange"}
    numbers = {1, 2, 3, 4, 5}
    mixed = {1, "hello", 3.14}
    
    # Empty set (must use set(), not {})
    empty_set = set()
    
    print(fruits)               # Output: {'banana', 'orange', 'apple'} (order may vary)
    print(type(fruits))         # Output: <class 'set'>


SET OPERATIONS:

    # Automatic duplicate removal
    numbers = {1, 2, 2, 3, 3, 3, 4}
    print(numbers)              # Output: {1, 2, 3, 4}
    
    # Adding items
    fruits = {"apple", "banana"}
    fruits.add("orange")
    print(fruits)               # Output: {'apple', 'banana', 'orange'}
    
    # Removing items
    fruits.remove("banana")     # Raises error if not found
    print(fruits)               # Output: {'apple', 'orange'}
    
    fruits.discard("grape")     # No error if not found
    
    # Checking membership (very fast!)
    print("apple" in fruits)    # Output: True
    
    # Set operations
    set1 = {1, 2, 3, 4}
    set2 = {3, 4, 5, 6}
    
    # Union (all items from both)
    print(set1 | set2)          # Output: {1, 2, 3, 4, 5, 6}
    
    # Intersection (common items)
    print(set1 & set2)          # Output: {3, 4}
    
    # Difference (items in set1 but not in set2)
    print(set1 - set2)          # Output: {1, 2}
    
    # Symmetric difference (items in either but not both)
    print(set1 ^ set2)          # Output: {1, 2, 5, 6}


CONVERTING LIST TO SET (REMOVE DUPLICATES):

    numbers_list = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4]
    unique_numbers = set(numbers_list)
    print(unique_numbers)       # Output: {1, 2, 3, 4}


REAL-WORLD USE CASES:
    - Removing duplicate emails from a list
    - Finding common friends on social media
    - Checking unique visitors to a website

8. DICTIONARY TYPE (dict)
================================================================================

WHAT: Collection of key-value pairs (like a real dictionary: word-definition)

WHY: Store related data with meaningful labels

HOW: Use curly braces {} with key:value pairs

Examples:

    # Creating dictionaries
    student = {
        "name": "John",
        "age": 20,
        "grade": "A"
    }
    
    prices = {
        "apple": 1.50,
        "banana": 0.75,
        "orange": 2.00
    }
    
    empty_dict = {}
    
    print(student)              # Output: {'name': 'John', 'age': 20, 'grade': 'A'}
    print(type(student))        # Output: <class 'dict'>


DICTIONARY OPERATIONS:

    # Accessing values
    student = {"name": "John", "age": 20, "grade": "A"}
    print(student["name"])      # Output: John
    print(student.get("age"))   # Output: 20
    
    # Adding/Modifying
    student["age"] = 21         # Modify existing
    student["city"] = "NYC"     # Add new key-value
    print(student)              # Output: {'name': 'John', 'age': 21, 'grade': 'A', 'city': 'NYC'}
    
    # Removing items
    del student["grade"]
    print(student)              # Output: {'name': 'John', 'age': 21, 'city': 'NYC'}
    
    removed_value = student.pop("city")
    print(removed_value)        # Output: NYC
    
    # Checking if key exists
    print("name" in student)    # Output: True
    print("grade" in student)   # Output: False
    
    # Getting all keys
    print(student.keys())       # Output: dict_keys(['name', 'age'])
    
    # Getting all values
    print(student.values())     # Output: dict_values(['John', 21])
    
    # Getting all items
    print(student.items())      # Output: dict_items([('name', 'John'), ('age', 21)])
    
    # Length
    print(len(student))         # Output: 2


NESTED DICTIONARIES:

    school = {
        "student1": {
            "name": "John",
            "age": 20
        },
        "student2": {
            "name": "Alice",
            "age": 22
        }
    }
    
    print(school["student1"]["name"])   # Output: John


REAL-WORLD USE CASES:
    - User profiles (username, email, password)
    - Product information (name, price, stock)
    - Configuration settings

9. BOOLEAN TYPE (bool)
================================================================================

WHAT: Represents True or False values

WHY: Used for decision making and conditions

HOW: Use True or False (capital first letter!)

Examples:

    # Boolean values
    is_student = True
    is_adult = False
    
    print(is_student)           # Output: True
    print(type(is_student))     # Output: <class 'bool'>


BOOLEAN FROM COMPARISONS:

    # Comparison operations return boolean
    age = 18
    print(age >= 18)            # Output: True
    print(age < 18)             # Output: False
    
    # Equality
    name = "John"
    print(name == "John")       # Output: True
    print(name == "Alice")      # Output: False


TRUTHY AND FALSY VALUES:

In Python, some values are considered "falsy" (treated as False):
    - False
    - None
    - 0 (zero)
    - 0.0
    - "" (empty string)
    - [] (empty list)
    - {} (empty dict)
    - () (empty tuple)
    - set() (empty set)

Everything else is "truthy" (treated as True)

Examples:

    # Using bool() to check
    print(bool(0))              # Output: False
    print(bool(1))              # Output: True
    print(bool(""))             # Output: False
    print(bool("hello"))        # Output: True
    print(bool([]))             # Output: False
    print(bool([1, 2]))         # Output: True


LOGICAL OPERATIONS:

    # AND operator
    print(True and True)        # Output: True
    print(True and False)       # Output: False
    
    # OR operator
    print(True or False)        # Output: True
    print(False or False)       # Output: False
    
    # NOT operator
    print(not True)             # Output: False
    print(not False)            # Output: True


REAL-WORLD USE CASES:
    - Checking if user is logged in
    - Validating form inputs
    - Game state (is_game_over)

10. TYPE CHECKING
================================================================================

WHAT: Finding out what type a variable is

WHY: Helps debug code and understand data

HOW: Use type() function

Examples:

    # Checking types
    age = 25
    name = "John"
    price = 19.99
    is_student = True
    fruits = ["apple", "banana"]
    
    print(type(age))            # Output: <class 'int'>
    print(type(name))           # Output: <class 'str'>
    print(type(price))          # Output: <class 'float'>
    print(type(is_student))     # Output: <class 'bool'>
    print(type(fruits))         # Output: <class 'list'>


USING isinstance():

    # Check if variable is of specific type
    age = 25
    print(isinstance(age, int))         # Output: True
    print(isinstance(age, str))         # Output: False
    
    # Check multiple types
    value = 3.14
    print(isinstance(value, (int, float)))  # Output: True


PRACTICAL USE:

    def process_data(data):
        if isinstance(data, str):
            print("Processing text:", data.upper())
        elif isinstance(data, int):
            print("Processing number:", data * 2)
        elif isinstance(data, list):
            print("Processing list:", len(data), "items")
    
    process_data("hello")       # Output: Processing text: HELLO
    process_data(10)            # Output: Processing number: 20
    process_data([1, 2, 3])     # Output: Processing list: 3 items

    # Int to string

12. MUTABLE VS IMMUTABLE
================================================================================

WHAT:
    - MUTABLE: Can be changed after creation
    - IMMUTABLE: Cannot be changed after creation

WHY: Understanding this prevents bugs and helps with memory management

--------------------------------------------------------------------------------
12.1 IMMUTABLE TYPES
--------------------------------------------------------------------------------

Cannot be changed after creation:
    - int
    - float
    - complex
    - str
    - tuple
    - bool
    - frozenset

Examples:

    # Strings are immutable
    name = "John"
    # name[0] = "K"             # TypeError: 'str' object does not support item assignment
    
    # To "change" a string, create a new one
    name = "K" + name[1:]
    print(name)                 # Output: Kohn
    
    # Numbers are immutable
    x = 10
    print(id(x))                # Memory address: e.g., 140234567890
    x = x + 5
    print(id(x))                # Different memory address!
    
    # Tuples are immutable
    point = (10, 20)
    # point[0] = 15             # TypeError: 'tuple' object does not support item assignment


--------------------------------------------------------------------------------
12.2 MUTABLE TYPES
--------------------------------------------------------------------------------

Can be changed after creation:
    - list
    - dict
    - set

Examples:

    # Lists are mutable
    fruits = ["apple", "banana"]
    print(id(fruits))           # Memory address: e.g., 140234567890
    
    fruits[0] = "orange"        # Modify in place
    print(fruits)               # Output: ['orange', 'banana']
    print(id(fruits))           # SAME memory address!
    
    # Dictionaries are mutable
    student = {"name": "John", "age": 20}
    student["age"] = 21         # Modify in place
    print(student)              # Output: {'name': 'John', 'age': 21}
    
    # Sets are mutable
    numbers = {1, 2, 3}
    numbers.add(4)              # Modify in place
    print(numbers)              # Output: {1, 2, 3, 4}


IMPORTANT CONCEPT - REFERENCE VS COPY:

    # With mutable objects, assignment creates reference
    list1 = [1, 2, 3]
    list2 = list1               # list2 points to SAME list
    
    list2.append(4)
    print(list1)                # Output: [1, 2, 3, 4] (CHANGED!)
    print(list2)                # Output: [1, 2, 3, 4]
    
    # To create independent copy
    list1 = [1, 2, 3]
    list2 = list1.copy()        # or list(list1) or list1[:]
    
    list2.append(4)
    print(list1)                # Output: [1, 2, 3] (unchanged)
    print(list2)                # Output: [1, 2, 3, 4]


DIAGRAM:

IMMUTABLE (String):
    name = "John"
    name = "Jane"
    
    Memory:
    [John] ← old (garbage collected)
    [Jane] ← new (name points here)


MUTABLE (List):
    fruits = ["apple"]
    fruits.append("banana")
    
    Memory:
    [apple, banana] ← same object, modified
    ↑
    fruits (still points here)


WHY IT MATTERS:

    # Function with mutable argument
    def add_item(shopping_list, item):
        shopping_list.append(item)
        return shopping_list
    
    my_list = ["apple", "banana"]
    result = add_item(my_list, "orange")
    
    print(my_list)              # Output: ['apple', 'banana', 'orange'] (MODIFIED!)
    print(result)               # Output: ['apple', 'banana', 'orange']
    
    # Both point to same list!

13. BASIC MEMORY UNDERSTANDING
================================================================================

WHAT: How Python stores data in computer memory

WHY: Helps understand variable behavior and write efficient code

--------------------------------------------------------------------------------
13.1 MEMORY ADDRESSES
--------------------------------------------------------------------------------

Every object in Python has a memory address (location in RAM)

    # Using id() to see memory address
    x = 10
    print(id(x))                # Output: 140234567890 (example)
    
    y = 10
    print(id(y))                # Output: 140234567890 (SAME!)
    
    # Python reuses small integers for efficiency


--------------------------------------------------------------------------------
13.2 VARIABLE AS LABELS
--------------------------------------------------------------------------------

Variables are like LABELS or NAME TAGS pointing to objects in memory

    x = 100
    
    Memory visualization:
    
    Variable     Memory
    --------     ------
    x     -----> [100]


MULTIPLE VARIABLES, SAME OBJECT:

    a = [1, 2, 3]
    b = a
    
    Memory visualization:
    
    Variable     Memory
    --------     ------
    a     -----> [1, 2, 3]
    b     ------^


REASSIGNMENT:

    x = 10
    x = 20
    
    Memory visualization:
    
    Before:
    x -----> [10]
    
    After:
    x -----> [20]
    [10] (garbage collected)


--------------------------------------------------------------------------------
13.3 MEMORY EFFICIENCY
--------------------------------------------------------------------------------

    # Small integers (-5 to 256) are cached
    a = 10
    b = 10
    print(a is b)               # Output: True (same object)
    
    # Large integers are not
    x = 1000
    y = 1000
    print(x is y)               # Output: False (different objects)
    
    # Strings are interned (cached) in some cases
    s1 = "hello"
    s2 = "hello"
    print(s1 is s2)             # Output: True


SIZE OF OBJECTS:

    import sys
    
    # Different types use different memory
    print(sys.getsizeof(10))            # Output: 28 bytes
    print(sys.getsizeof(10.5))          # Output: 24 bytes
    print(sys.getsizeof("hello"))       # Output: 54 bytes
    print(sys.getsizeof([1, 2, 3]))     # Output: 88 bytes

14. COMMON MISTAKES BEGINNERS MAKE
================================================================================

MISTAKE 1: Confusing strings and numbers

    # Wrong
    age = "25"
    next_year = age + 1         # TypeError!
    
    # Correct
    age = 25
    next_year = age + 1         # Output: 26


MISTAKE 2: Modifying immutable types

    # Wrong
    name = "john"
    name[0] = "J"               # TypeError!
    
    # Correct
    name = "J" + name[1:]       # Output: "John"


MISTAKE 3: Not understanding mutable references

    # Wrong assumption
    list1 = [1, 2, 3]
    list2 = list1
    list2.append(4)
    # Surprise! list1 is also [1, 2, 3, 4]
    
    # Correct
    list1 = [1, 2, 3]
    list2 = list1.copy()
    list2.append(4)
    # Now list1 is still [1, 2, 3]


MISTAKE 4: Using wrong type conversion

    # Wrong
    num = int("19.99")          # ValueError!
    
    # Correct
    num = int(float("19.99"))   # Output: 19


MISTAKE 5: Creating empty set incorrectly

    # Wrong
    empty = {}                  # This is a dict, not a set!
    
    # Correct
    empty = set()               # This is a set


MISTAKE 6: Forgetting tuple needs comma for single item

    # Wrong
    single = (42)               # This is just int 42
    print(type(single))         # Output: <class 'int'>
    
    # Correct
    single = (42,)              # This is a tuple
    print(type(single))         # Output: <class 'tuple'>


MISTAKE 7: Comparing with wrong operator

    # Wrong (assignment, not comparison)
    if age = 18:                # SyntaxError!
        print("Adult")
    
    # Correct
    if age == 18:
        print("Adult")


MISTAKE 8: Not checking type before operations

    # Risky
    def double(x):
        return x * 2
    
    print(double(5))            # Output: 10 (correct)
    print(double("5"))          # Output: "55" (unexpected!)
    
    # Better
    def double(x):
        if isinstance(x, (int, float)):
            return x * 2
        else:
            return "Error: Expected number"

15. TIPS FOR WRITING CLEAN PYTHON CODE
================================================================================

TIP 1: Use meaningful variable names

    # Bad
    x = "John"
    y = 25
    
    # Good
    name = "John"
    age = 25


TIP 2: Choose the right data type

    # Bad (using list for unique items)
    unique_ids = [101, 102, 103, 101]
    
    # Good (use set)
    unique_ids = {101, 102, 103}


TIP 3: Use type hints (Python 3.5+)

    def greet(name: str) -> str:
        return f"Hello, {name}"
    
    def add(a: int, b: int) -> int:
        return a + b


TIP 4: Convert types explicitly

    # Bad (implicit, unclear)
    age = "25"
    result = int(age) + 5
    
    # Good (explicit, clear)
    age_str = "25"
    age_int = int(age_str)
    result = age_int + 5


TIP 5: Use appropriate collection type

    # For ordered, changeable data
    shopping_list = ["apple", "banana", "orange"]
    
    # For ordered, unchangeable data
    coordinates = (10.5, 20.3)
    
    # For unique items
    unique_visitors = {"user1", "user2", "user3"}
    
    # For key-value pairs
    user_info = {"name": "John", "age": 25}


TIP 6: Check before converting

    def safe_int_convert(value):
        try:
            return int(value)
        except ValueError:
            return None
    
    result = safe_int_convert("123")    # Output: 123
    result = safe_int_convert("abc")    # Output: None


TIP 7: Use built-in functions

    # Instead of manual type checking
    numbers = [1, 2, 3, 4, 5]
    total = sum(numbers)            # Built-in sum()
    maximum = max(numbers)          # Built-in max()
    minimum = min(numbers)          # Built-in min()


TIP 8: Document expected types

    def calculate_area(length, width):
        """
        Calculate rectangle area.
        
        Args:
            length (float): Length of rectangle
            width (float): Width of rectangle
        
        Returns:
            float: Area of rectangle
        """
        return length * width

16. SUMMARY
================================================================================

KEY POINTS TO REMEMBER:

1. DATA TYPES classify data into categories (int, float, str, list, etc.)

2. NUMERIC TYPES:
   - int: whole numbers
   - float: decimal numbers
   - complex: complex numbers

3. TEXT TYPE:
   - str: text/characters

4. COLLECTION TYPES:
   - list: ordered, changeable [1, 2, 3]
   - tuple: ordered, unchangeable (1, 2, 3)
   - set: unordered, unique {1, 2, 3}
   - dict: key-value pairs {"name": "John"}

5. BOOLEAN TYPE:
   - bool: True or False

6. TYPE CHECKING:
   - type(): get type of variable
   - isinstance(): check if variable is specific type

7. TYPE CONVERSION:
   - int(), float(), str(), list(), tuple(), set(), dict()

8. MUTABLE (changeable):
   - list, dict, set

9. IMMUTABLE (unchangeable):
   - int, float, str, tuple, bool

10. CHOOSE RIGHT TYPE:
    - Use list for ordered, changeable collections
    - Use tuple for ordered, unchangeable collections
    - Use set for unique items
    - Use dict for labeled data
