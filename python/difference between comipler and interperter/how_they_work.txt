--------------------------------------------------------------------------------
                    HOW A COMPILER WORKS (DETAILED)
--------------------------------------------------------------------------------

A compiler goes through several stages to convert your code to machine code.

STAGE 1: LEXICAL ANALYSIS (SCANNING)
-------------------------------------

What happens:
- Compiler reads your source code character by character
- Breaks it into small pieces called "TOKENS"
- Tokens are like words in a sentence

Example:
Your code:    int x = 5;

Tokens created:
- "int"    (keyword)
- "x"      (identifier/variable name)
- "="      (operator)
- "5"      (number)
- ";"      (delimiter)

Think of it like: Breaking a sentence into individual words.


STAGE 2: SYNTAX ANALYSIS (PARSING)
-----------------------------------

What happens:
- Compiler checks if tokens follow grammar rules
- Creates a tree structure (Parse Tree)
- Verifies code structure is correct

Example:
Checks if:    int x = 5;    follows C language grammar rules

Correct:      int x = 5;
Wrong:        int = x 5;    (wrong order!)

Think of it like: Checking if sentence grammar is correct.


STAGE 3: SEMANTIC ANALYSIS
---------------------------

What happens:
- Compiler checks if code makes logical sense
- Verifies data types match
- Checks if variables are declared before use

Example:
Your code:    int x = "hello";

Problem: You're trying to put text ("hello") into a number variable (int)
Compiler error: Type mismatch!

Think of it like: Checking if sentence meaning makes sense.


STAGE 4: INTERMEDIATE CODE GENERATION
--------------------------------------

What happens:
- Compiler creates intermediate code
- This is between source code and machine code
- Easier to optimize

Example:
Your code:    x = a + b * c;

Intermediate code (simplified):
    temp1 = b * c
    x = a + temp1

Think of it like: Creating a rough draft before final version.


STAGE 5: CODE OPTIMIZATION
---------------------------

What happens:
- Compiler improves the code for better performance
- Removes unnecessary operations
- Makes code faster and smaller

Example:
Your code:
    x = 5;
    y = x + 0;    // Adding 0 does nothing!

Optimized:
    x = 5;
    y = x;        // Removed unnecessary +0

Think of it like: Editing to make writing more efficient.


STAGE 6: CODE GENERATION
-------------------------

What happens:
- Compiler converts optimized code to machine code
- Creates actual 0s and 1s
- Generates executable file

Example:
Final output: 01001000 01100101 01101100 01101100 01101111...

Think of it like: Creating the final published book.

--------------------------------------------------------------------------------
                  HOW AN INTERPRETER WORKS (DETAILED)
--------------------------------------------------------------------------------

An interpreter works differently - it processes code line-by-line.

STAGE 1: READ ONE LINE
-----------------------

What happens:
- Interpreter reads just ONE line of source code
- Keeps track of which line it's on

Example:
Reading line 1:    print("Hello")


STAGE 2: LEXICAL ANALYSIS (FOR THAT LINE)
------------------------------------------

What happens:
- Breaks that one line into tokens
- Same as compiler, but only for current line

Example:
Line:    print("Hello")

Tokens:
- "print"    (function name)
- "("        (opening parenthesis)
- "Hello"    (string)
- ")"        (closing parenthesis)


STAGE 3: SYNTAX ANALYSIS (FOR THAT LINE)
-----------------------------------------

What happens:
- Checks if that line follows grammar rules
- Creates parse tree for that line only

Example:
Checks:    print("Hello")    is valid Python syntax


STAGE 4: SEMANTIC ANALYSIS (FOR THAT LINE)
-------------------------------------------

What happens:
- Checks if that line makes logical sense
- Verifies variables exist (if used)
- Checks data types

Example:
If line is:    print(x)
Checks: Does variable 'x' exist?


STAGE 5: EXECUTE THAT LINE
---------------------------

What happens:
- Interpreter executes that line immediately
- Produces output if any
- Updates program state (variables, etc.)

Example:
Executes:    print("Hello")
Output:      Hello


STAGE 6: MOVE TO NEXT LINE
---------------------------

What happens:
- Interpreter moves to the next line
- Repeats the entire process
- Continues until program ends

--------------------------------------------------------------------------------
                    SIDE-BY-SIDE COMPARISON OF PROCESSES
--------------------------------------------------------------------------------

COMPILER PROCESS:
=================
1. Read ENTIRE source code
2. Analyze ALL code
3. Check ALL for errors
4. Optimize ALL code
5. Generate machine code for ALL
6. Create executable file
7. THEN user can run the executable
8. Execution is fast (already in machine code)


INTERPRETER PROCESS:
====================
1. Read ONE line
2. Analyze THAT line
3. Check THAT line for errors
4. Execute THAT line immediately
5. Move to NEXT line
6. Repeat steps 1-5
7. No executable file created
8. Execution is slower (translating while running)


--------------------------------------------------------------------------------
                    DETAILED EXAMPLE: BOTH APPROACHES
--------------------------------------------------------------------------------

Let's trace how both handle this simple program:

CODE:
-----
    Line 1: x = 5
    Line 2: y = 10
    Line 3: z = x + y
    Line 4: print(z)


COMPILER APPROACH (C-style):
=============================

COMPILATION PHASE:
------------------
Step 1: Read all 4 lines
Step 2: Create tokens for all lines
Step 3: Build parse tree for entire program
Step 4: Check all types and logic
Step 5: Generate intermediate code:
        - Assign 5 to x
        - Assign 10 to y
        - Add x and y, store in z
        - Print z
Step 6: Optimize (nothing to optimize here)
Step 7: Convert to machine code (binary)
Step 8: Save as program.exe

EXECUTION PHASE:
----------------
User runs: program.exe
- Machine code executes directly
- Output: 15
- Very fast execution


INTERPRETER APPROACH (Python-style):
=====================================

EXECUTION PHASE (No separate compilation):
-------------------------------------------
Step 1: Read Line 1: x = 5
        - Analyze: Valid syntax
        - Execute: Create variable x, assign 5
        - No output

Step 2: Read Line 2: y = 10
        - Analyze: Valid syntax
        - Execute: Create variable y, assign 10
        - No output

Step 3: Read Line 3: z = x + y
        - Analyze: Valid syntax
        - Check: Do x and y exist? Yes!
        - Execute: Add x (5) + y (10) = 15
        - Store in z
        - No output

Step 4: Read Line 4: print(z)
        - Analyze: Valid syntax
        - Check: Does z exist? Yes!
        - Execute: Print value of z
        - Output: 15

Program ends.


--------------------------------------------------------------------------------
                    WHAT HAPPENS WITH ERRORS?
--------------------------------------------------------------------------------

EXAMPLE CODE WITH ERROR:
------------------------
    Line 1: x = 5
    Line 2: y = 10
    Line 3: z = x + abc    // abc is not defined!
    Line 4: print(z)


COMPILER BEHAVIOR:
==================
Compilation Phase:
- Reads all lines
- Analyzes all lines
- Finds error on Line 3: "abc is not defined"
- Shows error message
- DOES NOT create executable
- Lines 1, 2, and 4 are NOT executed
- User must fix error and recompile


INTERPRETER BEHAVIOR:
=====================
Execution Phase:
- Reads and executes Line 1: x = 5 ✓
- Reads and executes Line 2: y = 10 ✓
- Reads Line 3: z = x + abc
- ERROR! "abc is not defined"
- Program STOPS here
- Line 4 is NEVER executed
- Lines 1 and 2 were successfully executed before error


--------------------------------------------------------------------------------
                    MEMORY USAGE DURING EXECUTION
--------------------------------------------------------------------------------

COMPILER:
=========
During Compilation:
- High memory usage
- Stores entire source code
- Stores parse trees
- Stores intermediate code
- Stores symbol tables

During Execution:
- Low memory usage
- Just runs machine code
- No translation overhead


INTERPRETER:
============
During Execution:
- Moderate memory usage
- Stores source code
- Stores current line info
- Stores variables and state
- Translation happens in memory
- Ongoing overhead


--------------------------------------------------------------------------------
                    TECHNICAL TERMS EXPLAINED
--------------------------------------------------------------------------------

TOKEN:
- Small piece of code (keyword, variable, operator, etc.)
- Example: In "int x = 5;", tokens are: int, x, =, 5, ;

PARSE TREE:
- Tree structure showing code grammar
- Shows how code is organized

SYNTAX:
- Grammar rules of the programming language
- Example: Statements must end with semicolon in C

SEMANTICS:
- Meaning and logic of the code
- Example: Can't assign text to integer variable

INTERMEDIATE CODE:
- Code between source and machine code
- Easier to optimize

MACHINE CODE:
- Binary code (0s and 1s) that CPU understands
- Example: 01001000 01100101...

OPTIMIZATION:
- Making code faster and more efficient
- Removing unnecessary operations

--------------------------------------------------------------------------------
                    KEY DIFFERENCES IN WORKING
--------------------------------------------------------------------------------

1. PROCESSING UNIT:
   Compiler: Processes entire program
   Interpreter: Processes one line at a time

2. ANALYSIS TIMING:
   Compiler: Analyzes all code before execution
   Interpreter: Analyzes code during execution

3. ERROR REPORTING:
   Compiler: Reports all errors together
   Interpreter: Reports errors as encountered

4. OUTPUT:
   Compiler: Creates executable file
   Interpreter: Executes directly, no file created

5. EXECUTION:
   Compiler: Fast (machine code runs directly)
   Interpreter: Slower (translation during runtime)

--------------------------------------------------------------------------------
                    REMEMBER THIS
--------------------------------------------------------------------------------

COMPILER:
Read ALL → Analyze ALL → Translate ALL → Create .exe → Run FAST

INTERPRETER:
Read Line → Analyze Line → Execute Line → Repeat → Run SLOWER
