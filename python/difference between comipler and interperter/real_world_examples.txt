--------------------------------------------------------------------------------
                    COMPILED LANGUAGES (EXAMPLES)
--------------------------------------------------------------------------------

These languages use compilers to translate code to machine code.

================================================================================
1. C LANGUAGE
================================================================================

TYPE: Compiled Language
COMPILER: GCC (GNU Compiler Collection), Clang, MSVC

HOW IT WORKS:
-------------
1. Write code in .c file
2. Compile using compiler
3. Get executable file
4. Run the executable


EXAMPLE CODE (hello.c):
-----------------------

    #include <stdio.h>
    
    int main() {
        printf("Hello from C!\n");
        return 0;
    }


COMPILATION AND EXECUTION:
--------------------------

Step 1: Save as hello.c

Step 2: Compile the code
        Command: gcc hello.c -o hello.exe
        
        What happens:
        - GCC compiler reads entire hello.c
        - Checks for syntax errors
        - Translates to machine code
        - Creates hello.exe (Windows) or hello (Linux/Mac)

Step 3: Run the executable
        Command: hello.exe (Windows) or ./hello (Linux/Mac)
        
        Output: Hello from C!

Step 4: Run again
        Command: hello.exe
        
        Output: Hello from C!
        (No compilation needed - just run the .exe)


WHY C USES COMPILER:
--------------------
- C is used for system programming (operating systems, drivers)
- Speed is critical
- Direct hardware access needed
- Compiled code runs very fast


================================================================================
2. C++ LANGUAGE
================================================================================

TYPE: Compiled Language
COMPILER: G++, Clang++, MSVC

HOW IT WORKS:
-------------
Same as C, but with object-oriented features


EXAMPLE CODE (calculator.cpp):
-------------------------------

    #include <iostream>
    using namespace std;
    
    int main() {
        int a = 10;
        int b = 5;
        int sum = a + b;
        
        cout << "Sum: " << sum << endl;
        return 0;
    }


COMPILATION AND EXECUTION:
--------------------------

Step 1: Save as calculator.cpp

Step 2: Compile
        Command: g++ calculator.cpp -o calculator.exe
        
        What happens:
        - G++ reads entire file
        - Checks syntax and types
        - Translates to machine code
        - Creates calculator.exe

Step 3: Run
        Command: calculator.exe
        Output: Sum: 15


WHY C++ USES COMPILER:
----------------------
- Used for games, applications, system software
- Performance is crucial
- Compiled code is very fast
- Good for resource-intensive applications


================================================================================
3. RUST LANGUAGE
================================================================================

TYPE: Compiled Language
COMPILER: rustc (Rust Compiler)

HOW IT WORKS:
-------------
Modern compiled language with safety features


EXAMPLE CODE (greet.rs):
------------------------

    fn main() {
        let name = "World";
        println!("Hello, {}!", name);
    }


COMPILATION AND EXECUTION:
--------------------------

Step 1: Save as greet.rs

Step 2: Compile
        Command: rustc greet.rs
        
        What happens:
        - Rust compiler checks code
        - Ensures memory safety
        - Translates to machine code
        - Creates greet.exe or greet

Step 3: Run
        Command: greet.exe
        Output: Hello, World!


WHY RUST USES COMPILER:
-----------------------
- System programming language
- Memory safety is critical
- Performance like C/C++
- Compiled for speed and safety


--------------------------------------------------------------------------------
                    INTERPRETED LANGUAGES (EXAMPLES)
--------------------------------------------------------------------------------

These languages use interpreters to execute code line-by-line.

================================================================================
1. PYTHON LANGUAGE
================================================================================

TYPE: Interpreted Language
INTERPRETER: CPython, PyPy

HOW IT WORKS:
-------------
1. Write code in .py file
2. Run using Python interpreter
3. Code executes line-by-line
4. No executable file created


EXAMPLE CODE (hello.py):
------------------------

    print("Hello from Python!")
    print("This is line 2")
    print("This is line 3")


EXECUTION:
----------

Step 1: Save as hello.py

Step 2: Run using interpreter
        Command: python hello.py
        
        What happens:
        - Python interpreter reads line 1
        - Executes: print("Hello from Python!")
        - Output: Hello from Python!
        - Reads line 2
        - Executes: print("This is line 2")
        - Output: This is line 2
        - Reads line 3
        - Executes: print("This is line 3")
        - Output: This is line 3
        - Program ends

Step 3: Run again
        Command: python hello.py
        
        Same process repeats
        (Interpretation happens every time)


WHY PYTHON USES INTERPRETER:
-----------------------------
- Easy to learn and use
- Quick development
- Great for scripting and automation
- Flexibility is more important than speed
- Excellent for data science, web development


EXAMPLE WITH VARIABLES (calculator.py):
----------------------------------------

    a = 10
    b = 5
    sum = a + b
    print("Sum:", sum)


EXECUTION PROCESS:
------------------
Line 1: a = 10
        - Interpreter creates variable a
        - Assigns value 10
        
Line 2: b = 5
        - Interpreter creates variable b
        - Assigns value 5
        
Line 3: sum = a + b
        - Interpreter calculates 10 + 5 = 15
        - Creates variable sum
        - Assigns value 15
        
Line 4: print("Sum:", sum)
        - Interpreter prints: Sum: 15


================================================================================
2. JAVASCRIPT LANGUAGE
================================================================================

TYPE: Interpreted Language (with JIT compilation in modern engines)
INTERPRETER: V8 (Chrome), SpiderMonkey (Firefox)

HOW IT WORKS:
-------------
Runs in web browsers or Node.js environment


EXAMPLE CODE (script.js):
--------------------------

    console.log("Hello from JavaScript!");
    
    let x = 5;
    let y = 10;
    let result = x + y;
    
    console.log("Result:", result);


EXECUTION IN BROWSER:
---------------------

Step 1: Include in HTML file
        <script src="script.js"></script>

Step 2: Browser's JavaScript engine executes
        - Reads line 1: console.log("Hello from JavaScript!")
        - Outputs to console: Hello from JavaScript!
        - Reads line 3: let x = 5;
        - Creates variable x with value 5
        - Reads line 4: let y = 10;
        - Creates variable y with value 10
        - Reads line 5: let result = x + y;
        - Calculates and stores 15
        - Reads line 7: console.log("Result:", result)
        - Outputs: Result: 15


EXECUTION IN NODE.JS:
---------------------

Command: node script.js

Same interpretation process as browser


WHY JAVASCRIPT USES INTERPRETER:
---------------------------------
- Runs in web browsers
- Dynamic and flexible
- Code can be modified at runtime
- Quick development for web applications
- Modern engines use JIT for better performance

================================================================================

--------------------------------------------------------------------------------
                    HYBRID LANGUAGES (BOTH APPROACHES)
--------------------------------------------------------------------------------

Some languages use BOTH compilation and interpretation!

================================================================================
1. JAVA LANGUAGE
================================================================================

TYPE: Hybrid (Compiled to Bytecode, then Interpreted/JIT Compiled)
TOOLS: javac (compiler), JVM (Java Virtual Machine)

HOW IT WORKS:
-------------
1. Source code compiled to bytecode
2. Bytecode is interpreted by JVM
3. JIT compiler optimizes during runtime


EXAMPLE CODE (Hello.java):
---------------------------

    public class Hello {
        public static void main(String[] args) {
            System.out.println("Hello from Java!");
        }
    }


COMPILATION AND EXECUTION:
--------------------------

Step 1: Save as Hello.java

Step 2: Compile to bytecode
        Command: javac Hello.java
        
        What happens:
        - javac compiler reads Hello.java
        - Checks syntax and types
        - Translates to bytecode (not machine code!)
        - Creates Hello.class file
        
        Note: Bytecode is NOT machine code
              It's an intermediate format

Step 3: Run using JVM
        Command: java Hello
        
        What happens:
        - JVM reads Hello.class (bytecode)
        - Interprets bytecode
        - JIT compiler optimizes hot code
        - Output: Hello from Java!


VISUAL FLOW:
------------

    Hello.java (Source Code)
            |
            | javac (Compiler)
            v
    Hello.class (Bytecode)
            |
            | JVM (Interpreter + JIT)
            v
    Machine Code (at runtime)
            |
            v
    Output


WHY JAVA USES HYBRID APPROACH:
-------------------------------
- Write Once, Run Anywhere (WORA)
- Bytecode runs on any platform with JVM
- JIT compilation provides good performance
- Balance between portability and speed


ADVANTAGES:
-----------
- Portable (same bytecode runs on Windows, Mac, Linux)
- Faster than pure interpretation
- Safer than direct compilation
- Platform-independent


================================================================================
2. PYTHON (WITH BYTECODE)
================================================================================

TYPE: Primarily Interpreted, but uses bytecode internally
TOOLS: CPython interpreter

HOW IT ACTUALLY WORKS:
----------------------
Most people think Python is purely interpreted, but:

1. Python source code (.py) is compiled to bytecode (.pyc)
2. Bytecode is interpreted by Python Virtual Machine (PVM)
3. This happens automatically and invisibly


EXAMPLE CODE (test.py):
-----------------------

    def greet(name):
        return f"Hello, {name}!"
    
    print(greet("World"))


WHAT REALLY HAPPENS:
--------------------

Step 1: Run command
        Command: python test.py

Step 2: Python compiles to bytecode
        - Creates test.pyc (in __pycache__ folder)
        - This is bytecode, not machine code
        - Happens automatically

Step 3: PVM interprets bytecode
        - Executes line-by-line
        - Output: Hello, World!

Step 4: Next time you run
        - If test.py hasn't changed, uses cached test.pyc
        - Faster startup


VISUAL FLOW:
------------

    test.py (Source Code)
            |
            | Automatic compilation
            v
    test.pyc (Bytecode)
            |
            | PVM (Interpreter)
            v
    Execution
            |
            v
    Output


WHY THIS HYBRID APPROACH:
--------------------------
- Faster than interpreting source code directly
- Still flexible like interpreted languages
- Bytecode is platform-independent
- Best of both worlds


================================================================================
3. C# LANGUAGE
================================================================================

TYPE: Hybrid (Compiled to IL, then JIT compiled)
TOOLS: C# Compiler, .NET Runtime

HOW IT WORKS:
-------------
Similar to Java


EXAMPLE CODE (Program.cs):
---------------------------

    using System;
    
    class Program {
        static void Main() {
            Console.WriteLine("Hello from C#!");
        }
    }


COMPILATION AND EXECUTION:
--------------------------

Step 1: Compile to IL (Intermediate Language)
        Command: csc Program.cs
        Creates: Program.exe (contains IL, not machine code)

Step 2: Run using .NET Runtime
        Command: Program.exe
        
        What happens:
        - .NET Runtime reads IL
        - JIT compiler converts IL to machine code
        - Executes machine code
        - Output: Hello from C#!

--------------------------------------------------------------------------------
                    PRACTICAL COMPARISON EXAMPLE
--------------------------------------------------------------------------------

Let's write the SAME program in different languages:

TASK: Calculate sum of numbers from 1 to 1000000


C VERSION (compiled.c):
-----------------------

    #include <stdio.h>
    
    int main() {
        long sum = 0;
        for(int i = 1; i <= 1000000; i++) {
            sum += i;
        }
        printf("Sum: %ld\n", sum);
        return 0;
    }

Compile: gcc compiled.c -o compiled.exe
Run: compiled.exe
Time: ~0.01 seconds (VERY FAST!)


PYTHON VERSION (interpreted.py):
---------------------------------

    sum = 0
    for i in range(1, 1000001):
        sum += i
    print(f"Sum: {sum}")

Run: python interpreted.py
Time: ~0.1 seconds (slower, but still acceptable)


JAVA VERSION (hybrid.java):
----------------------------

    public class Hybrid {
        public static void main(String[] args) {
            long sum = 0;
            for(int i = 1; i <= 1000000; i++) {
                sum += i;
            }
            System.out.println("Sum: " + sum);
        }
    }

Compile: javac Hybrid.java
Run: java Hybrid
Time: ~0.02 seconds (middle ground)


PERFORMANCE COMPARISON:
-----------------------
C (Compiled):        0.01 seconds  ★★★★★ (Fastest)
Java (Hybrid):       0.02 seconds  ★★★★☆
Python (Interpreted): 0.1 seconds  ★★★☆☆


--------------------------------------------------------------------------------
                    WHEN TO USE WHICH LANGUAGE?
--------------------------------------------------------------------------------

USE COMPILED LANGUAGES (C, C++, Rust) WHEN:
-------------------------------------------
- Maximum performance needed
- System-level programming
- Game development
- Embedded systems
- Operating systems
- Device drivers


USE INTERPRETED LANGUAGES (Python, JavaScript, Ruby) WHEN:
----------------------------------------------------------
- Quick development needed
- Scripting and automation
- Web development
- Data analysis
- Prototyping
- Learning programming


USE HYBRID LANGUAGES (Java, C#) WHEN:
--------------------------------------
- Need portability across platforms
- Enterprise applications
- Android apps (Java/Kotlin)
- Windows applications (C#)
- Balance between speed and flexibility


--------------------------------------------------------------------------------
                    KEY TAKEAWAYS
--------------------------------------------------------------------------------

1. C, C++, Rust = Pure compilation (fastest execution)

2. Python, JavaScript, Ruby = Interpretation (easiest development)

3. Java, C# = Hybrid approach (balanced)

4. Choice depends on your project needs

5. Modern languages often use hybrid approaches

6. Performance vs Development Speed is the main trade-off

--------------------------------------------------------------------------------
                    REMEMBER THIS
--------------------------------------------------------------------------------

COMPILED (C, C++):     Write → Compile → Fast Execution
INTERPRETED (Python):  Write → Run Immediately → Flexible
HYBRID (Java):         Write → Compile to Bytecode → Run Anywhere
