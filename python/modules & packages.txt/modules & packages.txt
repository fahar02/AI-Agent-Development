## BASIC CONCEPTS

### What is a Module?

A module is a single Python file (`.py`) that contains:
- Functions
- Classes  
- Variables
- Constants
- Executable code

**Simple Definition**: Module = File containing reusable code

**Example Files**:
- `math.py` - Mathematical functions
- `os.py` - Operating system interface
- `user.py` - User-related functionality
- `database.py` - Database operations

### What is a Package?

A package is a directory that contains multiple modules and optionally sub-packages.

**Simple Definition**: Package = Collection of related modules

**Example Structure**:
```
my_package/
├── __init__.py          # Makes directory a package
├── module1.py           # First module
├── module2.py           # Second module
├── subpackage/          # Sub-package
│   ├── __init__.py
│   └── module3.py
└── utils.py             # Utility functions
```

## WHY USE MODULES

### 1. Code Reusability
```python
# calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

```

### 2. Better Organization
```
project/
├── auth/           # Authentication functions
├── database/       # Database operations
├── utils/          # Utility functions
├── models/         # Data models
└── views/          # User interface
```

### 3. Avoid Code Duplication
- Write once, use many times
- Single source of truth
- Easier maintenance

### 4. Namespace Separation
```python
# math.py
def sqrt(x):
    return x ** 0.5

# custom_math.py
def sqrt(x):
    return x * x  # Different implementation

import math
import custom_math

print(math.sqrt(16))        # 4.0
print(custom_math.sqrt(16)) # 256
```

### 5. Easier Maintenance
- Small, focused files
- Clear responsibilities
- Easier debugging
- Better testing

## IMPORTING MODULES

### 1. Basic Import
```python
import math

# Usage with module name prefix
result = math.sqrt(16)
print(result)  # 4.0
```

### 2. Import Specific Items
```python

# Direct usage (no prefix)
```

### 3. Import with Alias
```python

```

## HOW IMPORT WORKS INTERNALLY

### The Import Process

When Python executes `import module_name`:

1. **Check Cache**: Look in `sys.modules` dictionary
2. **Search**: If not cached, search in `sys.path`
3. **Load**: Find and load the module file
4. **Compile**: Compile to bytecode if needed
5. **Execute**: Execute module code
6. **Cache**: Store in `sys.modules`

### Detailed Example

```python
import sys
import importlib

# View module cache
print("Cached modules:", len(sys.modules))

# Import a module
import math

# Check if it's cached
print("math in cache:", 'math' in sys.modules)

# View module object
print("Math module:", math)
print("Math module file:", math.__file__)
```

### Module Loading Steps

```python
# Step 1: Check cache
if 'my_module' not in sys.modules:
    # Step 2: Search in sys.path
    for path in sys.path:
        module_path = os.path.join(path, 'my_module.py')
        if os.path.exists(module_path):
            # Step 3: Load and execute
            with open(module_path) as f:
                code = compile(f.read(), module_path, 'exec')
                exec(code, module_dict)
            # Step 4: Cache the module
            sys.modules['my_module'] = module_dict
            break
```

## MODULE SEARCH PATH

### Understanding `sys.path`

```python
import sys
print("Module search paths:")
for i, path in enumerate(sys.path, 1):
    print(f"{i}. {path}")
```

### Typical `sys.path` Contents

1. **Current Directory**: Where the script is running
2. **PYTHONPATH**: Environment variable paths
3. **Standard Library**: Built-in modules location
4. **Site Packages**: Third-party installed packages

### Modifying Search Path

```python
import sys

# Add custom directory
sys.path.append('/path/to/my/modules')

# Add to beginning (higher priority)
sys.path.insert(0, '/path/to/priority/modules')

# Temporary modification
with sys.path_context('/path/to/temp/modules'):
    import my_module  # Will search here first
```

### Finding Module Location

```python
import os
import math

# Find where module is located
print("os module location:", os.__file__)
print("math module location:", math.__file__)

# Check if module is built-in
print("math is built-in:", math.__file__.startswith('<'))
```

## CREATING CUSTOM MODULES

### Step 1: Create Module File

**`my_module.py`**:
```python
"""
My Custom Module
================

This module provides utility functions for mathematical operations.
"""

# Module-level variables
PI = 3.14159265359
VERSION = "1.0.0"

# Functions
def circle_area(radius):
    """Calculate area of a circle."""
    return PI * radius ** 2

def circle_circumference(radius):
    """Calculate circumference of a circle."""
    return 2 * PI * radius

# Classes
class Circle:
    """Circle class with radius and methods."""
    
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return circle_area(self.radius)
    
    def circumference(self):
        return circle_circumference(self.radius)
    
    def __str__(self):
        return f"Circle(radius={self.radius})"

# Module initialization code
print(f"MyModule v{VERSION} loaded successfully!")
```

### Step 2: Use the Module

**`main.py`**:
```python
import my_module

# Use module variables
print("PI value:", my_module.PI)
print("Module version:", my_module.VERSION)

# Use functions
area = my_module.circle_area(5)
print("Circle area:", area)

# Use classes
circle = my_module.Circle(5)
print(circle)  # Circle(radius=5)
print("Area:", circle.area())
```

## UNDERSTANDING `__name__`

### The `__name__` Variable

Each module has a special variable `__name__`:

- `"__main__"` → When file is run directly
- Module name → When imported

### Basic Example

**`test_module.py`**:
```python
def greet(name):
    return f"Hello, {name}!"

def main():
    print("This is running directly!")
    print(greet("World"))

if __name__ == "__main__":
    main()
else:
    print("This module was imported")
```

### Behavior Demonstration

```bash
# Run directly
$ python test_module.py
This is running directly!
Hello, World!

# Import from another file
>>> import test_module
This module was imported
>>> test_module.greet("Alice")
'Hello, Alice!'
```

### Practical Uses

#### 1. Testing Code
```python
def calculate_area(length, width):
    return length * width

if __name__ == "__main__":
    # Test the function
    assert calculate_area(5, 3) == 15
    assert calculate_area(10, 2) == 20
    print("All tests passed!")
```

#### 2. Command-Line Interface
```python
import sys

def process_data(data):
    return data.upper()

if __name__ == "__main__":
    if len(sys.argv) > 1:
        input_data = sys.argv[1]
        print(process_data(input_data))
    else:
        print("Usage: python script.py <data>")
```

#### 3. Module Configuration
```python
import logging

# Configure logging only when run directly
if __name__ == "__main__":
    logging.basicConfig(level=logging.DEBUG)
    logger = logging.getLogger(__name__)
    logger.info("Module running in debug mode")
```

## PACKAGES OVERVIEW

### What Makes a Package?

A directory becomes a package when it contains an `__init__.py` file.

**Simple Package Structure**:
```
my_package/
├── __init__.py          # Required! Makes it a package
├── module1.py
├── module2.py
└── utils.py
```

### The `__init__.py` File

#### Minimal `__init__.py`
```python
# Empty file - just marks directory as package
```

#### Basic `__init__.py`
```python
"""
My Package
==========

A collection of useful utilities.
"""

__version__ = "1.0.0"
__author__ = "Your Name"
```

#### Advanced `__init__.py`
```python
# Control what gets imported
from .module1 import function1, Class1
from .module2 import function2
from .utils import helper_function

# Package-level variables
PACKAGE_NAME = "my_package"
VERSION = "1.0.0"

# Package initialization
print(f"{PACKAGE_NAME} v{VERSION} loaded")
```

### Package Benefits

1. **Organization**: Group related modules
2. **Namespace**: Avoid naming conflicts
3. **Distribution**: Easy to share and install
4. **Hierarchy**: Create nested packages

## RELATIVE VS ABSOLUTE IMPORTS

### Absolute Imports

**Definition**: Import from the project root using full path.

```python
# ecommerce/products/views.py
from ecommerce.auth.login import authenticate_user
from ecommerce.utils.validators import validate_email
from ecommerce.cart.cart_manager import CartManager
```

**Advantages**:
- Clear and explicit
- Easy to understand
- Works from anywhere
- Better for large projects

### Relative Imports

**Definition**: Import relative to current package location.

```python
# ecommerce/products/views.py
from ..auth.login import authenticate_user      # Parent package
from ..utils.validators import validate_email  # Parent package
from .models import Product                    # Same package
from .controllers import ProductController     # Same package
```

**Relative Import Rules**:
- `.` : Current package
- `..` : Parent package
- `...` : Grandparent package
- etc.

### When to Use Each

#### Use Absolute Imports When:
- Project structure is stable
- You want clarity and explicitness
- Working in large codebases
- Refactoring frequently

#### Use Relative Imports When:
- Package is self-contained
- You might rename the package
- Working within a single package
- Package might be moved

### Best Practice Example

```python
# ecommerce/products/views.py

# Absolute imports (recommended for most cases)
from ecommerce.auth.login import authenticate_user
from ecommerce.cart.manager import CartManager

# Relative imports (for internal package references)
from .models import Product, Category
from .controllers import ProductController
```

## MODULE CACHING

### Understanding `sys.modules`

Python caches imported modules in `sys.modules` dictionary:

```python
import sys

# View cached modules
print(f"Total cached modules: {len(sys.modules)}")

# Check if specific module is cached
print("math cached:", 'math' in sys.modules)
print("json cached:", 'json' in sys.modules)

# Access cached module
cached_math = sys.modules['math']
print("Cached math sqrt:", cached_math.sqrt(9))
```

### How Caching Works

```python
# First import - loads and caches
import my_module
print("First import - module loaded")

# Second import - uses cache (no reload)
import my_module
print("Second import - from cache")

# Both references point to same object
import my_module as module1
import my_module as module2

print("Same object:", module1 is module2)  # True
```

### Benefits of Caching

1. **Performance**: Faster subsequent imports
2. **Memory Efficiency**: Single instance per module
3. **State Preservation**: Module-level variables persist
4. **Consistency**: Same object throughout application

### Cache Demonstration

```python
# counter_module.py
counter = 0

def increment():
    global counter
    counter += 1
    return counter

# main.py
import counter_module

print(counter_module.increment())  # 1
print(counter_module.increment())  # 2

# Reimport (still same cached module)
import counter_module
print(counter_module.increment())  # 3 (continues from previous state)
```

## ADVANCED IMPORT TECHNIQUES

### 1. Dynamic Import

```python
import importlib

# Import module by name string
module_name = "math"
math_module = importlib.import_module(module_name)
result = math_module.sqrt(16)

# Import from module by string
from_module = importlib.import_module("json.loads")
loads_function = getattr(from_module, "loads")
data = loads_function('{"key": "value"}')
```

### 2. Conditional Import

```python
try:
    import pandas as pd
except ImportError:
    print("pandas not available, using alternative")
    import csv as pd

# Platform-specific imports
import sys
if sys.platform == "win32":
    import windows_specific as platform_utils
else:
    import unix_specific as platform_utils
```

### 3. Lazy Import

```python
class LazyImport:
    def __init__(self, module_name):
        self.module_name = module_name
        self._module = None
    
    def __getattr__(self, name):
        if self._module is None:
            self._module = importlib.import_module(self.module_name)
        return getattr(self._module, name)

# Usage
heavy_library = LazyImport("heavy_computation_library")
# Library not loaded until actually used
result = heavy_library.some_function()
```

### 4. Import Hooks and Custom Loaders

```python
import importlib.abc
import importlib.util

class CustomLoader(importlib.abc.Loader):
    def create_module(self, spec):
        return None  # Use default module creation
    
    def exec_module(self, module):
        # Custom module loading logic
        module.custom_attribute = "Loaded by custom loader"

# Register custom loader
custom_loader = CustomLoader()
spec = importlib.util.spec_from_loader("custom_module", custom_loader)
custom_module = importlib.util.module_from_spec(spec)
spec.loader.exec_module(custom_module)
```

### 5. Module Reloading

```python
import importlib
import my_module

# Make changes to my_module.py while program is running

# Reload the module
importlib.reload(my_module)

# Now my_module has updated code
```

## NAMESPACE MANAGEMENT

### Understanding Namespaces

A namespace is a mapping from names to objects.

```python
# Different namespaces
x = 10           # Global namespace

def my_func():
    x = 20       # Local namespace
    print(x)     # Prints 20

print(x)         # Prints 10
my_func()        # Prints 20
```

### Module Namespaces

```python
# math module namespace
import math
print(math.pi)   # pi from math namespace

# Create our own pi
pi = 3.14
print(pi)        # Our pi from global namespace
print(math.pi)   # Still math.pi
```

### Avoiding Name Conflicts

```python
# Potential conflict
from statistics import mean
from numpy import mean  # Overwrites the previous mean!

# Solution: Use module prefix
import statistics
import numpy
print(statistics.mean([1, 2, 3]))
print(numpy.mean([1, 2, 3]))

# Or use aliases
from statistics import mean as stat_mean
from numpy import mean as np_mean
```

### Namespace Hierarchy

```python
# Built-in namespace
print(len([1, 2, 3]))  # Built-in len function

# Global namespace
len = "custom value"
print(len)  # Global variable

# Local namespace
def my_function():
    len = 5  # Local variable
    print(len)

my_function()
```

### Cleaning Namespaces

```python
# Remove from namespace
import math
del math  # Remove from current namespace

# Clear all variables (use with caution!)
# globals().clear()
```

## COMMON MISTAKES

### 1. Circular Imports

**Problem**:
```python
# module_a.py
import module_b

def func_a():
    module_b.func_b()

# module_b.py  
import module_a

def func_b():
    module_a.func_a()
```

**Solution**:
```python
# module_a.py
def func_a():
    from module_b import func_b  # Import inside function
    func_b()

# module_b.py
def func_b():
    from module_a import func_a  # Import inside function
    func_a()
```

### 2. Missing `__init__.py`

**Problem**:
```
my_package/
├── module1.py  # Can't import as package
└── module2.py
```

**Solution**:
```
my_package/
├── __init__.py  # Empty file
├── module1.py
└── module2.py
```

### 3. Name Conflicts

**Problem**:
```python
from math import sqrt
from my_module import sqrt  # Overwrites previous sqrt
```

**Solution**:
```python
import math
from my_module import sqrt as custom_sqrt
```

### 4. Importing in Functions

**Problem**:
```python
def process_data():
    import pandas  # Repeated import on every call
    return pandas.DataFrame()
```

**Solution**:
```python
import pandas  # Import at module level

def process_data():
    return pandas.DataFrame()
```

### 5. Mutable Default Arguments

**Problem**:
```python
# module.py
def add_item(item, items=[]):  # Dangerous!
    items.append(item)
    return items
```

**Solution**:
```python
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

### 3. Metaprogramming with Modules

```python
# Dynamic module creation
import types

def create_module(name, functions):
    """Create a module dynamically."""
    module = types.ModuleType(name)
    
    for func_name, func_code in functions.items():
        setattr(module, func_name, func_code)
    
    return module

# Usage
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

math_module = create_module("custom_math", {
    "add": add,
    "multiply": multiply
})

print(math_module.add(5, 3))      # 8
print(math_module.multiply(4, 2)) # 8
```

### 4. Module Loading Hooks

```python
import sys
import importlib.util

def custom_import_hook(name):
    """Custom import hook for special modules."""
    if name.startswith("custom_"):
        # Custom loading logic
        spec = importlib.util.spec_from_loader(name, loader=None)
        module = importlib.util.module_from_spec(spec)

    return None

# Register hook (simplified example)
```

### 5. Module Performance Optimization

```python
# Lazy loading for heavy modules
class LazyModule:
    def __init__(self, module_name):
        self._module_name = module_name
        self._module = None
    
    def __getattr__(self, name):
        if self._module is None:
            import importlib
            self._module = importlib.import_module(self._module_name)
        return getattr(self._module, name)

# Usage
numpy = LazyModule("numpy")
# numpy not loaded until first use
array = numpy.array([1, 2, 3])
```

## FINAL SUMMARY

### Modules
- **Definition**: Single Python file (`.py`)
- **Purpose**: Code organization and reusability
- **Key Features**: Functions, classes, variables
- **Benefits**: Maintainability, testing, collaboration

### Packages
- **Definition**: Directory with `__init__.py` containing modules
- **Purpose**: Organize related modules
- **Key Features**: Hierarchical structure, namespace management
- **Benefits**: Scalability, distribution, clarity

### Golden Rules

1. **One module, one responsibility**
2. **Use packages for large projects**
3. **Avoid circular imports**
4. **Prefer absolute imports**
5. **Understand `__name__` behavior**
6. **Use meaningful names**
7. **Document your modules**
8. **Test your modules independently**
