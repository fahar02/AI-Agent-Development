WHAT IS COMPREHENSION?

Comprehension is a short and clean way to create collections like:

List
Set
Dictionary

It replaces traditional loops with a single line of code.

In simple terms:
Comprehension = Loop + Condition + Output in one line

WHY USE COMPREHENSIONS?
Less code (concise)
More readable (once you understand)
Faster than normal loops (optimized internally)
BASIC STRUCTURE

[expression for item in iterable]

Example:
squares = [x*x for x in range(5)]

Output:
[0, 1, 4, 9, 16]

LIST COMPREHENSION (BEGINNER)

4.1 Normal Loop
numbers = []
for i in range(5):
numbers.append(i)

4.2 Using List Comprehension
numbers = [i for i in range(5)]

ADDING CONDITION

5.1 Syntax:
[expression for item in iterable if condition]

Example:
even_numbers = [x for x in range(10) if x % 2 == 0]

Output:
[0, 2, 4, 6, 8]

IF-ELSE IN COMPREHENSION

Syntax:
[expression_if_true if condition else expression_if_false for item in iterable]

Example:
result = ["Even" if x % 2 == 0 else "Odd" for x in range(5)]

Output:
["Even", "Odd", "Even", "Odd", "Even"]

NESTED LOOPS

Syntax:
[expression for i in iterable1 for j in iterable2]

Example:
pairs = [(i, j) for i in range(2) for j in range(2)]

Output:
[(0,0), (0,1), (1,0), (1,1)]

NESTED COMPREHENSION (ADVANCED)

Example:
matrix = [[j for j in range(3)] for i in range(3)]

Output:
[[0,1,2], [0,1,2], [0,1,2]]

SET COMPREHENSION

Syntax:
{expression for item in iterable}

Example:
unique_squares = {x*x for x in [1,2,2,3]}

Output:
{1, 4, 9}

(Note: Set removes duplicates)

DICTIONARY COMPREHENSION

Syntax:
{key:value for item in iterable}

Example:
square_dict = {x: x*x for x in range(5)}

Output:
{0:0, 1:1, 2:4, 3:9, 4:16}

ADD CONDITION IN DICT

Example:
even_square_dict = {x: x*x for x in range(10) if x % 2 == 0}

MULTIPLE CONDITIONS

Example:
result = [x for x in range(20) if x % 2 == 0 if x % 3 == 0]

Equivalent:
if x % 2 == 0 and x % 3 == 0

USING FUNCTIONS

Example:
def square(x):
return x*x

result = [square(x) for x in range(5)]

STRING COMPREHENSION

Example:
chars = [c.upper() for c in "hello"]

Output:
['H', 'E', 'L', 'L', 'O']

FLATTENING LISTS (ADVANCED)

Example:
matrix = [[1,2,3], [4,5,6]]

flat = [num for row in matrix for num in row]

Output:
[1,2,3,4,5,6]

PERFORMANCE

List comprehension is:

Faster than loops (in most cases)
Uses less memory than traditional methods

But:

Avoid very complex nested comprehensions (reduces readability)
GENERATOR COMPREHENSION (IMPORTANT)

Syntax:
(expression for item in iterable)

Example:
gen = (x*x for x in range(5))

Difference:

Uses parentheses ()
Does NOT store all values in memory
Generates values one by one

Useful for large data

COMMON MISTAKES
Forgetting brackets:
[x for x in range(5)] ✔
x for x in range(5) ❌
Wrong order:
[for x in range(5) x] ❌
Over-complicating logic (avoid deep nesting)
WHEN TO USE / NOT USE

Use:

Simple transformations
Filtering data
Creating lists quickly

Avoid:

Very complex logic
Multiple nested conditions (hard to read)
REAL-WORLD EXAMPLES

Example 1:
Get emails from users
emails = [user.email for user in users]

Example 2:
Filter active users
active_users = [u for u in users if u.is_active]

Example 3:
API data transformation
ids = [item["id"] for item in response]

INTERVIEW INSIGHT (IMPORTANT)

Interviewers expect:

Basic syntax
Condition usage
Nested comprehension understanding
Difference between list and generator
FINAL SUMMARY

Comprehension is:

A powerful shortcut for loops
Clean and Pythonic way to write code
Essential for writing efficient and readable programs

Golden Rule:
"Use comprehension for simplicity, not complexity"
