#python architecture
The main components of Python tools that we install are known as the **Python Development Environment**.
The most important part is the **Python Interpreter**, which includes:
- The **Python Compiler** (converts `.py` to bytecode '.pyc' )
- The **Python Virtual Machine (PVM)** (executes bytecode)
- The **Interpreter itself** (runs your Python programs)


#How Python Program Executes
Whenever we create a Python file with the extension .py, it is called the source code or Python code.
When we execute this code, it is passed to the Python interpreter.
The first step inside the interpreter is the Python compiler, which converts the source code into bytecode, stored with a .pyc extension in the  __pycache__ folder.
This bytecode is then sent to the Python Virtual Machine (PVM).
The PVM executes the bytecode line by line, using the interpreter logic built into it.
During execution, the PVM also loads any imported libraries or modules needed by the program.

ever time we execute the code we get the  __pycache__ folder. which store the byte code as use a caching of the code 
during executing. On future runs, Python uses the .pyc bytecode instead of re-compiling the .py file for faster startup.

# A variable in Python is a reference to a memory location. 
# Python is a dynamically-typed language, so we do not need to explicitly define the data type.
# The Python interpreter automatically detects the type based on the value assigned.

# There are four types of variable scopes in Python, following the LEGB rule:
# - Local
# - Enclosed
# - Global
# - Built-in
# - instance
# - class

# 1. Local Variable:
# A variable that is declared inside a function.
# It is only accessible within that function.

def func():
    x = 10  # Local variable
    print(x)

# 2. Enclosed Variable:
# Python supports nested functions (a function defined inside another function).
# An enclosed variable is one that exists in the outer (enclosing) function, and is accessible by the inner function.

def outer():
    name = "Fahar"  # Enclosed variable
    def inner():
        print(name)  # Accessing enclosed variable
    inner()

# 3. Global Variable:
# A variable that is defined at the top-level of a Python script (outside any function).
# It can be accessed from any function within the same module.
# To modify a global variable inside a function, the 'global' keyword is used.

x = 5  # Global variable

def show():
    global x
    x = 100
    print(x)

# 4. Built-in Variable:
# These are predefined names in Python, such as len, print, list, etc.
# These come from Python’s built-in namespace.
# Overriding them should be avoided to prevent unexpected behavior.

print(len("Hello"))  # 'len' is a built-in function

#variable shadow and resolution:
Variable shadowing occurs when a variable in an inner scope has the same name as one in an outer scope. The inner variable “shadows” the outer one in that scope. For example:
In this code, the local a = 5 inside func() has the same name as the global a. Python treats them as completely separate variables. The local a shadows the global one within func(). As GeeksforGeeks notes: “the global variable a is said to be shadowed by the local variable a.”
geeksforgeeks.orgIn other words, references to a inside func() resolve to the local version, not the global. Shadowing can lead to confusion or subtle bugs if unintended. To force the use of the outer name instead, use global or nonlocal. For example, to modify the global a within func, write global a in func. To modify a variable in an enclosing function, use nonlocal. 



#Scope Resolution: Python always follows LEGB order, so a local name will override (shadow) outer names. If a name is not found in the local scope, Python searches enclosing scopes, then global, then built-ins
realpython.com If no declaration (global/nonlocal) is used, assigning to a name in a function always creates a new local variable, which is why forgetting global leads to an UnboundLocalError.

# shadow of the class and instance variable:
If both class and instance have a variable with the same name, the instance variable takes priority. Python will use the instance variable if it exists; otherwise, it uses the class variable.

int data type:
The int data type is used to store integer values (whole numbers without decimals).
In Python, int is actually a class (i.e., int is a built-in type implemented as a class).
You can use the int() function to convert other types (like strings or floats) into integers.
The default value of an integer variable is 0 if not explicitly assigned.

float Data Type in Python:
The float data type is used to store floating-point numbers (decimal values like 3.14, -0.5).
In Python, float is actually a class – that means all float values are objects of the built-in float class.
You can use the float() function to convert other types (like integers or strings) into float values.
The default value of a float (when using float() without arguments) is 0.0.
While converting a string using float(), the string must represent a valid float, otherwise it raises a ValueError.

bool Data Type in Python
The bool (Boolean) data type is used to store logical values: True or False.
Internally, True is represented as 1 and False as 0, but they are not the same as integers — they are instances of the built-in bool class, which is a subclass of int.
This means:
True behaves like the integer 1
False behaves like the integer 0
You can use the bool() function to convert any value into its Boolean form, following Python's truthiness rules.

Functions in Python – Access Modifiers and Scope
Functions are blocks of code or sets of instructions used to perform a specific task.
Python does not have strict access modifiers like Java (public, private, protected), but it uses naming conventions to indicate the intended scope or accessibility of a function.
There are three main types of function accessibility based on naming style:
1. Public Functions (Default)
Any function that does not start with an underscore is considered public.
It can be accessed from: Within the same class or module, Subclasses, Other modules
This is the default behavior in Python.
2. Protected Functions
Functions whose names start with a single underscore (_) are treated as protected.
This is only a convention — the function is still accessible outside the class/module, but it's intended for internal or subclass use only.
Used to indicate: "Don't use this unless you're extending the class."
3. Private Functions
Functions starting with double underscores (__) are treated as private.
Python performs name mangling — it internally renames the function to _ClassName__functionname.This makes it difficult (but not impossible) to access from outside the class.

#Precedence / Order of Function Parameters
ote: After *args, all parameters must be passed as keyword arguments, otherwise you’ll get an error.
The / means everything before it must be positional
/ → Positional-Only Parameters
Parameters before / can only be passed by position, not by name.
You must give them without specifying parameter name in function call.

The * means everything after it must be keyword-only
* → Keyword-Only Parameters
Parameters after * can only be passed by name (as keyword arguments).
Even though there’s no *args, the * forces keyword-only behavior.

def func(a, b, /, c, d=4, *, e, f=6):
    print(f"a = {a}, b = {b}, c = {c}, d = {d}, e = {e}, f = {f}")
Explanation:
Parameter	How It Must Be Passed	Notes
a, b	Positional only	Must pass as func(1, 2, ...)
c, d	Positional or keyword	You can pass c=3, or 3
e, f	Keyword only (after *)	Must pass as e=value
