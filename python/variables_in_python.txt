1. WHAT IS A VARIABLE?
================================================================================

A variable is like a CONTAINER or BOX that stores data in computer memory.

REAL-LIFE ANALOGY:
Think of a variable like a labeled box in your room:
- The LABEL on the box is the variable name (e.g., "age")
- The ITEM inside the box is the value (e.g., 25)
- You can change what's inside the box anytime

Example:

    age = 25

Here:
- "age" is the variable name (the label)
- "=" is the assignment operator (puts value in the box)
- "25" is the value (what's stored in the box)


SIMPLE DEFINITION:
A variable is a NAME that refers to a VALUE stored in computer memory.


================================================================================
2. WHY VARIABLES ARE IMPORTANT
================================================================================

Variables are essential because they:

1. STORE DATA: Keep information for later use
2. MAKE CODE READABLE: "age" is clearer than just "25"
3. REUSE VALUES: Use the same value multiple times
4. UPDATE DATA: Change values as needed
5. PERFORM CALCULATIONS: Work with data dynamically

Example WITHOUT variables (BAD):

    print("Hello, John")
    print("John is 25 years old")
    print("John lives in New York")
    # If name changes, you must update 3 places!


Example WITH variables (GOOD):

    name = "John"
    age = 25
    city = "New York"
    
    print("Hello,", name)
    print(name, "is", age, "years old")
    print(name, "lives in", city)
    # If name changes, update only 1 place!

3. RULES FOR NAMING VARIABLES
================================================================================

Python has STRICT rules and BEST PRACTICES for variable names.

--------------------------------------------------------------------------------
MANDATORY RULES (Must Follow):
--------------------------------------------------------------------------------

RULE 1: Must start with a letter (a-z, A-Z) or underscore (_)

    # Valid
    name = "John"
    _age = 25
    user1 = "Alice"
    
    # Invalid
    1user = "Bob"          # SyntaxError: cannot start with number
    @name = "Test"         # SyntaxError: cannot use special characters


RULE 2: Can only contain letters, numbers, and underscores

    # Valid
    user_name = "John"
    age2 = 30
    _private_var = 100
    
    # Invalid
    user-name = "John"     # SyntaxError: hyphen not allowed
    user.name = "John"     # SyntaxError: dot not allowed
    user name = "John"     # SyntaxError: space not allowed


RULE 3: Case-sensitive (age and Age are different)

    age = 25
    Age = 30
    AGE = 35
    
    print(age)             # Output: 25
    print(Age)             # Output: 30
    print(AGE)             # Output: 35
    # All three are DIFFERENT variables!


RULE 4: Cannot use Python keywords (reserved words)

    # Invalid (these are Python keywords)
    if = 10                # SyntaxError
    for = 20               # SyntaxError
    class = "Math"         # SyntaxError
    def = 100              # SyntaxError

Python Keywords (Cannot Use):
    False, None, True, and, as, assert, async, await, break, class,
    continue, def, del, elif, else, except, finally, for, from, global,
    if, import, in, is, lambda, nonlocal, not, or, pass, raise, return,
    try, while, with, yield


--------------------------------------------------------------------------------
BEST PRACTICES (Should Follow):
--------------------------------------------------------------------------------

PRACTICE 1: Use descriptive names

    # Bad
    x = "John"
    y = 25
    z = "New York"
    
    # Good
    name = "John"
    age = 25
    city = "New York"


PRACTICE 2: Use lowercase with underscores (snake_case)

    # Good (Python convention)
    user_name = "John"
    total_price = 99.99
    is_student = True
    
    # Avoid (not Python style)
    userName = "John"      # camelCase (used in Java, JavaScript)
    UserName = "John"      # PascalCase (used for class names)


PRACTICE 3: Use UPPERCASE for constants

    # Constants (values that shouldn't change)
    PI = 3.14159
    MAX_USERS = 100
    DATABASE_URL = "localhost:5432"


PRACTICE 4: Avoid single-letter names (except in loops)

    # Bad
    a = 100
    b = 200
    c = a + b
    
    # Good
    price = 100
    tax = 200
    total = price + tax
    
    # Exception: single letters OK in short loops
    for i in range(10):
        print(i)


PRACTICE 5: Don't use built-in function names

    # Bad (overwrites built-in functions)
    list = [1, 2, 3]       # Now you can't use list() function!
    sum = 100              # Now you can't use sum() function!
    
    # Good
    numbers_list = [1, 2, 3]
    total_sum = 100

4. CREATING VARIABLES (ASSIGNMENT)
================================================================================

In Python, you create a variable by ASSIGNING a value to it.

SYNTAX:
    variable_name = value

Examples:

    # Integer
    age = 25
    
    # Float
    price = 19.99
    
    # String
    name = "John"
    
    # Boolean
    is_student = True
    
    # List
    fruits = ["apple", "banana", "orange"]
    
    # Dictionary
    person = {"name": "John", "age": 25}


NO DECLARATION NEEDED:

Unlike some languages (C, Java), Python doesn't require declaration:

    # In C or Java, you must declare type first:
    # int age;
    # age = 25;
    
    # In Python, just assign:
    age = 25               # Variable created automatically!


REASSIGNMENT (Changing Values):

    age = 25
    print(age)             # Output: 25
    
    age = 30               # Reassign new value
    print(age)             # Output: 30
    
    age = "thirty"         # Can even change type!
    print(age)             # Output: thirty

5. DYNAMIC TYPING IN PYTHON
================================================================================

WHAT: Python is DYNAMICALLY TYPED - variables can change type during execution

WHY: Provides flexibility but requires careful coding

HOW: Python determines type automatically based on the value

Examples:

    # Variable starts as integer
    x = 10
    print(type(x))         # Output: <class 'int'>
    
    # Same variable becomes string
    x = "hello"
    print(type(x))         # Output: <class 'str'>
    
    # Same variable becomes list
    x = [1, 2, 3]
    print(type(x))         # Output: <class 'list'>


COMPARISON WITH STATIC TYPING:

    # In statically-typed languages (like Java):
    # int age = 25;
    # age = "thirty";      // ERROR! Cannot change type
    
    # In Python (dynamically-typed):
    age = 25
    age = "thirty"         # Perfectly fine!


ADVANTAGES:
    - More flexible
    - Less code to write
    - Easier for beginners

DISADVANTAGES:
    - Can lead to unexpected errors
    - Harder to catch type-related bugs
    - May need more testing


PRACTICAL EXAMPLE:

    def calculate_total(price, quantity):
        return price * quantity
    
    # Works with numbers
    print(calculate_total(10, 5))      # Output: 50
    
    # Also works with strings (but unexpected!)
    print(calculate_total("Hi", 3))    # Output: HiHiHi
    
    # This is why type checking is important!

6. MULTIPLE ASSIGNMENT
================================================================================

Python allows assigning values to multiple variables in one line.

--------------------------------------------------------------------------------
6.1 SAME VALUE TO MULTIPLE VARIABLES
--------------------------------------------------------------------------------

    # Assign same value to multiple variables
    x = y = z = 0
    
    print(x)               # Output: 0
    print(y)               # Output: 0
    print(z)               # Output: 0


PRACTICAL USE:

    # Initialize multiple counters
    count1 = count2 = count3 = 0
    
    # Initialize coordinates
    x = y = z = 0.0


--------------------------------------------------------------------------------
6.2 DIFFERENT VALUES TO MULTIPLE VARIABLES
--------------------------------------------------------------------------------

    # Assign different values in one line
    name, age, city = "John", 25, "New York"
    
    print(name)            # Output: John
    print(age)             # Output: 25
    print(city)            # Output: New York


MORE EXAMPLES:

    # Coordinates
    x, y, z = 10, 20, 30
    
    # RGB colors
    red, green, blue = 255, 0, 128
    
    # Boolean flags
    is_active, is_admin, is_verified = True, False, True


--------------------------------------------------------------------------------
6.3 UNPACKING SEQUENCES
--------------------------------------------------------------------------------

    # Unpacking a list
    fruits = ["apple", "banana", "orange"]
    fruit1, fruit2, fruit3 = fruits
    
    print(fruit1)          # Output: apple
    print(fruit2)          # Output: banana
    print(fruit3)          # Output: orange
    
    # Unpacking a tuple
    coordinates = (10, 20, 30)
    x, y, z = coordinates
    
    print(x)               # Output: 10
    print(y)               # Output: 20
    print(z)               # Output: 30


USING ASTERISK (*) FOR REMAINING VALUES:

    # Get first, second, and rest
    numbers = [1, 2, 3, 4, 5, 6]
    first, second, *rest = numbers
    
    print(first)           # Output: 1
    print(second)          # Output: 2
    print(rest)            # Output: [3, 4, 5, 6]
    
    # Get first, rest, and last
    first, *middle, last = numbers
    
    print(first)           # Output: 1
    print(middle)          # Output: [2, 3, 4, 5]
    print(last)            # Output: 6


PRACTICAL EXAMPLE:

    # Function returning multiple values
    def get_user_info():
        return "John", 25, "New York"
    
    name, age, city = get_user_info()
    print(name)            # Output: John
    print(age)             # Output: 25
    print(city)            # Output: New York

7. VARIABLE SWAPPING
================================================================================

Swapping means exchanging values between variables.

--------------------------------------------------------------------------------
7.1 TRADITIONAL METHOD (Other Languages)
--------------------------------------------------------------------------------

    # Using temporary variable
    a = 10
    b = 20
    
    temp = a               # Store a in temp
    a = b                  # Put b in a
    b = temp               # Put temp (old a) in b
    
    print(a)               # Output: 20
    print(b)               # Output: 10


--------------------------------------------------------------------------------
7.2 PYTHON'S ELEGANT METHOD
--------------------------------------------------------------------------------

    # Swap in one line!
    a = 10
    b = 20
    
    a, b = b, a            # Swap values
    
    print(a)               # Output: 20
    print(b)               # Output: 10


HOW IT WORKS:

    # Python evaluates right side first
    a, b = b, a
    
    Step 1: Right side creates tuple (b, a) = (20, 10)
    Step 2: Left side unpacks tuple to a, b
    Result: a = 20, b = 10


MORE EXAMPLES:

    # Swap three variables
    x, y, z = 1, 2, 3
    x, y, z = z, y, x
    print(x, y, z)         # Output: 3 2 1
    
    # Swap in a list
    numbers = [10, 20, 30]
    numbers[0], numbers[2] = numbers[2], numbers[0]
    print(numbers)         # Output: [30, 20, 10]


PRACTICAL USE:

    # Sorting two numbers
    a = 50
    b = 30
    
    if a > b:
        a, b = b, a        # Ensure a <= b
    
    print(a, b)            # Output: 30 50

8. CONSTANTS CONCEPT
================================================================================

WHAT: Constants are variables whose values should NOT change

WHY: Makes code clearer and prevents accidental changes

HOW: Python doesn't have true constants, but we use UPPERCASE names as convention

Examples:

    # Constants (by convention)
    PI = 3.14159
    MAX_USERS = 100
    MIN_AGE = 18
    DATABASE_URL = "localhost:5432"
    API_KEY = "abc123xyz"
    
    # Use constants
    radius = 5
    area = PI * radius * radius
    print(area)            # Output: 78.53975


IMPORTANT NOTE:

Python doesn't enforce constants - you CAN change them, but you SHOULDN'T:

    PI = 3.14159
    PI = 3.14              # Python allows this, but DON'T do it!


BENEFITS OF CONSTANTS:

1. READABILITY: "MAX_USERS" is clearer than "100"
2. MAINTAINABILITY: Change value in one place
3. PREVENT ERRORS: Less likely to accidentally change

Example:

    # Without constants (BAD)
    if users > 100:
        print("Too many users")
    if connections > 100:
        print("Too many connections")
    # If limit changes, update multiple places!
    
    # With constants (GOOD)
    MAX_LIMIT = 100
    
    if users > MAX_LIMIT:
        print("Too many users")
    if connections > MAX_LIMIT:
        print("Too many connections")
    # If limit changes, update only one place!


GROUPING CONSTANTS:

    # Configuration constants
    DB_HOST = "localhost"
    DB_PORT = 5432
    DB_NAME = "myapp"
    
    # Application constants
    MAX_LOGIN_ATTEMPTS = 3
    SESSION_TIMEOUT = 3600
    DEFAULT_LANGUAGE = "en"

9. SCOPE OF VARIABLES
================================================================================

WHAT: Scope determines WHERE a variable can be accessed in your code

WHY: Understanding scope prevents bugs and naming conflicts

TYPES OF SCOPE:
    1. Local Scope
    2. Enclosing Scope (in nested functions)
    3. Global Scope
    4. Built-in Scope

DIAGRAM:

    Scope Hierarchy (LEGB Rule):
    
    +----------------------------------+
    |  Built-in Scope                  |  (print, len, etc.)
    |  +----------------------------+  |
    |  |  Global Scope              |  |  (module level)
    |  |  +----------------------+  |  |
    |  |  |  Enclosing Scope     |  |  |  (outer function)
    |  |  |  +----------------+  |  |  |
    |  |  |  | Local Scope    |  |  |  |  (inner function)
    |  |  |  +----------------+  |  |  |
    |  |  +----------------------+  |  |
    |  +----------------------------+  |
    +----------------------------------+

10. LOCAL SCOPE
================================================================================

WHAT: Variables created INSIDE a function

WHY: Keeps variables isolated to where they're needed

LIFETIME: Exists only while function is executing

Examples:

    def greet():
        message = "Hello"      # Local variable
        print(message)
    
    greet()                    # Output: Hello
    print(message)             # NameError: message not defined


DETAILED EXAMPLE:

    def calculate_area(length, width):
        area = length * width  # Local variable
        return area
    
    result = calculate_area(5, 10)
    print(result)              # Output: 50
    print(area)                # NameError: area not defined
    
    # 'area' only exists inside the function!


MULTIPLE FUNCTIONS, SAME VARIABLE NAME:

    def function1():
        x = 10                 # Local to function1
        print("Function1:", x)
    
    def function2():
        x = 20                 # Different x, local to function2
        print("Function2:", x)
    
    function1()                # Output: Function1: 10
    function2()                # Output: Function2: 20
    
    # Each function has its own 'x'


PARAMETERS ARE LOCAL:

    def add(a, b):             # a and b are local variables
        result = a + b         # result is also local
        return result
    
    total = add(5, 10)
    print(total)               # Output: 15
    print(a)                   # NameError: a not defined


================================================================================
11. GLOBAL SCOPE
================================================================================

WHAT: Variables created OUTSIDE all functions (at module level)

WHY: Accessible from anywhere in the file

LIFETIME: Exists for entire program execution

Examples:

    # Global variable
    name = "John"
    
    def greet():
        print("Hello,", name)  # Can access global variable
    
    def farewell():
        print("Goodbye,", name)  # Can access global variable
    
    greet()                    # Output: Hello, John
    farewell()                 # Output: Goodbye, John


READING GLOBAL VARIABLES:

    count = 0                  # Global variable
    
    def show_count():
        print(count)           # Reading global variable (OK)
    
    show_count()               # Output: 0


MODIFYING GLOBAL VARIABLES (REQUIRES 'global' KEYWORD):

    count = 0                  # Global variable
    
    def increment():
        global count           # Declare we're using global variable
        count = count + 1      # Modify global variable
    
    print(count)               # Output: 0
    increment()
    print(count)               # Output: 1
    increment()
    print(count)               # Output: 2


WITHOUT 'global' KEYWORD (CREATES LOCAL VARIABLE):

    count = 0                  # Global variable
    
    def increment():
        count = count + 1      # ERROR! UnboundLocalError
        # Python sees assignment, thinks count is local
        # But tries to read it before assignment
    
    increment()                # UnboundLocalError


CORRECT USAGE:

    total = 0                  # Global variable
    
    def add_to_total(amount):
        global total           # Declare global
        total += amount        # Modify global
    
    print(total)               # Output: 0
    add_to_total(10)
    print(total)               # Output: 10
    add_to_total(5)
    print(total)               # Output: 15


MULTIPLE GLOBAL VARIABLES:

    x = 10
    y = 20
    
    def modify_both():
        global x, y            # Declare multiple globals
        x = 100
        y = 200
    
    print(x, y)                # Output: 10 20
    modify_both()
    print(x, y)                # Output: 100 200


WHEN TO USE GLOBAL VARIABLES:

GOOD USE CASES:
    - Configuration settings
    - Constants
    - Shared counters/state

BAD USE CASES:
    - Passing data between functions (use parameters instead)
    - Temporary calculations

12. NONLOCAL SCOPE
================================================================================

WHAT: Variables in enclosing (outer) function, accessed from nested (inner) function

WHY: Allows inner functions to modify outer function variables

HOW: Use 'nonlocal' keyword

Examples:

    def outer():
        count = 0              # Enclosing scope variable
        
        def inner():
            nonlocal count     # Declare nonlocal
            count += 1         # Modify enclosing variable
            print("Inner count:", count)
        
        inner()
        inner()
        print("Outer count:", count)
    
    outer()
    # Output:
    # Inner count: 1
    # Inner count: 2
    # Outer count: 2


WITHOUT 'nonlocal' (CREATES LOCAL VARIABLE):

    def outer():
        count = 0
        
        def inner():
            count = 1          # Creates NEW local variable
            print("Inner count:", count)
        
        inner()
        print("Outer count:", count)
    
    outer()
    # Output:
    # Inner count: 1
    # Outer count: 0  (unchanged!)


PRACTICAL EXAMPLE (CLOSURE):

    def make_counter():
        count = 0
        
        def increment():
            nonlocal count
            count += 1
            return count
        
        return increment
    
    counter1 = make_counter()
    print(counter1())          # Output: 1
    print(counter1())          # Output: 2
    print(counter1())          # Output: 3
    
    counter2 = make_counter()  # Independent counter
    print(counter2())          # Output: 1
    print(counter2())          # Output: 2


NESTED LEVELS:

    def outer():
        x = 10
        
        def middle():
            x = 20             # Shadows outer's x
            
            def inner():
                nonlocal x     # Refers to middle's x
                x = 30
                print("Inner x:", x)
            
            inner()
            print("Middle x:", x)
        
        middle()
        print("Outer x:", x)
    
    outer()
    # Output:
    # Inner x: 30
    # Middle x: 30
    # Outer x: 10

13. LEGB RULE EXPLAINED
================================================================================

WHAT: LEGB is the order Python searches for variables

L - Local (inside current function)
E - Enclosing (in outer functions)
G - Global (module level)
B - Built-in (Python's built-in names)

HOW IT WORKS:

When you use a variable, Python searches in this order:
    1. Local scope (current function)
    2. Enclosing scope (outer functions)
    3. Global scope (module level)
    4. Built-in scope (Python built-ins)

If not found in any scope: NameError


EXAMPLE DEMONSTRATING LEGB:

    x = "global"               # Global scope
    
    def outer():
        x = "enclosing"        # Enclosing scope
        
        def inner():
            x = "local"        # Local scope
            print(x)
        
        inner()
    
    outer()                    # Output: local
    
    # Python found 'x' in Local scope first


SEARCHING THROUGH SCOPES:

    x = "global"
    
    def outer():
        # No x here (Enclosing)
        
        def inner():
            # No x here (Local)
            print(x)           # Searches: Local → Enclosing → Global (found!)
        
        inner()
    
    outer()                    # Output: global


BUILT-IN SCOPE EXAMPLE:

    # 'len' is a built-in function
    print(len([1, 2, 3]))      # Output: 3
    
    # You can shadow built-ins (but don't!)
    def my_function():
        len = 10               # Local variable shadows built-in
        print(len)             # Output: 10 (local, not built-in)
    
    my_function()
    print(len([1, 2, 3]))      # Output: 3 (built-in still works outside)


COMPLETE LEGB EXAMPLE:

    x = "built-in would be here"  # Global
    
    def outer():
        x = "enclosing"           # Enclosing
        
        def middle():
            x = "middle-enclosing"  # Enclosing for inner
            
            def inner():
                x = "local"       # Local
                print("1:", x)    # Finds in Local
            
            inner()
            print("2:", x)        # Finds in Local (middle's)
        
        middle()
        print("3:", x)            # Finds in Local (outer's)
    
    outer()
    print("4:", x)                # Finds in Global
    
    # Output:
    # 1: local
    # 2: middle-enclosing
    # 3: enclosing
    # 4: built-in would be here


PRACTICAL IMPLICATIONS:

    # Be careful with variable names!
    list = [1, 2, 3]           # Shadows built-in list()
    
    # Now list() function doesn't work!
    # new_list = list("abc")   # TypeError!
    
    # Solution: use different name
    my_list = [1, 2, 3]        # Good!

14. MEMORY REFERENCE CONCEPT
================================================================================

WHAT: Variables are REFERENCES (pointers) to objects in memory

WHY: Understanding this prevents common bugs

HOW: Variables don't store values directly; they point to memory locations

--------------------------------------------------------------------------------
14.1 BASIC CONCEPT
--------------------------------------------------------------------------------

    x = 10
    
    Memory Visualization:
    
    Variable     Memory Address     Value
    --------     --------------     -----
    x       ---> 0x7fff1234       [  10  ]


CHECKING MEMORY ADDRESS:

    x = 10
    print(id(x))               # Output: 140234567890 (example)
    
    # id() returns memory address


--------------------------------------------------------------------------------
14.2 MULTIPLE VARIABLES, SAME OBJECT
--------------------------------------------------------------------------------

    a = 10
    b = a                      # b points to SAME object as a
    
    print(id(a))               # Output: 140234567890
    print(id(b))               # Output: 140234567890 (SAME!)
    
    Memory Visualization:
    
    Variable     Memory
    --------     ------
    a       ---> [  10  ]
    b       -----^


FOR IMMUTABLE TYPES (int, str, tuple):

    x = 10
    y = x
    
    y = 20                     # Creates NEW object
    
    print(x)                   # Output: 10 (unchanged)
    print(y)                   # Output: 20
    
    print(id(x))               # Different address
    print(id(y))               # Different address


FOR MUTABLE TYPES (list, dict, set):

    list1 = [1, 2, 3]
    list2 = list1              # Both point to SAME list
    
    list2.append(4)            # Modifies the shared list
    
    print(list1)               # Output: [1, 2, 3, 4] (CHANGED!)
    print(list2)               # Output: [1, 2, 3, 4]
    
    print(id(list1))           # Same address
    print(id(list2))           # Same address
    
    Memory Visualization:
    
    Variable     Memory
    --------     ------
    list1   ---> [1, 2, 3, 4]
    list2   -----^


CREATING INDEPENDENT COPY:

    list1 = [1, 2, 3]
    list2 = list1.copy()       # Creates NEW list
    
    list2.append(4)
    
    print(list1)               # Output: [1, 2, 3] (unchanged)
    print(list2)               # Output: [1, 2, 3, 4]
    
    print(id(list1))           # Different address
    print(id(list2))           # Different address


--------------------------------------------------------------------------------
14.3 FUNCTION ARGUMENTS
--------------------------------------------------------------------------------

IMMUTABLE ARGUMENTS (int, str, tuple):

    def modify(x):
        x = 100                # Creates new local variable
        print("Inside:", x)
    
    num = 10
    modify(num)                # Output: Inside: 100
    print("Outside:", num)     # Output: Outside: 10 (unchanged)


MUTABLE ARGUMENTS (list, dict, set):

    def modify(lst):
        lst.append(4)          # Modifies original list!
        print("Inside:", lst)
    
    numbers = [1, 2, 3]
    modify(numbers)            # Output: Inside: [1, 2, 3, 4]
    print("Outside:", numbers) # Output: Outside: [1, 2, 3, 4] (CHANGED!)


AVOIDING UNINTENDED MODIFICATION:

    def modify(lst):
        lst = lst.copy()       # Work with copy
        lst.append(4)
        return lst
    
    numbers = [1, 2, 3]
    result = modify(numbers)
    
    print("Original:", numbers)  # Output: Original: [1, 2, 3] (unchanged)
    print("Result:", result)     # Output: Result: [1, 2, 3, 4]


--------------------------------------------------------------------------------
14.4 IDENTITY VS EQUALITY
--------------------------------------------------------------------------------

'is' checks if two variables point to SAME object (identity)
'==' checks if two variables have SAME value (equality)

    # Same object
    a = [1, 2, 3]
    b = a
    
    print(a == b)              # Output: True (same value)
    print(a is b)              # Output: True (same object)
    
    # Different objects, same value
    a = [1, 2, 3]
    b = [1, 2, 3]
    
    print(a == b)              # Output: True (same value)
    print(a is b)              # Output: False (different objects)


SPECIAL CASE (Small integers and strings):

    # Python caches small integers (-5 to 256)
    a = 10
    b = 10
    print(a is b)              # Output: True (same object!)
    
    # Large integers are not cached
    a = 1000
    b = 1000
    print(a is b)              # Output: False (different objects)
    
    # Strings are often interned
    a = "hello"
    b = "hello"
    print(a is b)              # Output: True (same object!)

15. COMMON MISTAKES
================================================================================

MISTAKE 1: Using undefined variable

    print(age)                 # NameError: name 'age' is not defined
    
    # Fix: Define before use
    age = 25
    print(age)                 # Output: 25


MISTAKE 2: Typo in variable name

    user_name = "John"
    print(username)            # NameError: name 'username' is not defined
    
    # Python is case-sensitive and exact!


MISTAKE 3: Using Python keywords

    class = "Math"             # SyntaxError: invalid syntax
    
    # Fix: Use different name
    class_name = "Math"


MISTAKE 4: Forgetting 'global' keyword

    count = 0
    
    def increment():
        count = count + 1      # UnboundLocalError!
    
    # Fix: Use global
    def increment():
        global count
        count = count + 1


MISTAKE 5: Unintended global modification

    total = 0
    
    def calculate():
        global total           # Modifies global!
        total = 100
    
    calculate()
    print(total)               # Output: 100 (global changed!)
    
    # Better: Use return value
    def calculate():
        return 100
    
    total = calculate()


MISTAKE 6: Mutable default arguments

    def add_item(item, lst=[]):  # BAD! Default list is shared
        lst.append(item)
        return lst
    
    print(add_item(1))         # Output: [1]
    print(add_item(2))         # Output: [1, 2] (Unexpected!)
    
    # Fix: Use None as default
    def add_item(item, lst=None):
        if lst is None:
            lst = []
        lst.append(item)
        return lst
    
    print(add_item(1))         # Output: [1]
    print(add_item(2))         # Output: [2] (Correct!)


MISTAKE 7: Modifying list while iterating

    numbers = [1, 2, 3, 4, 5]
    
    for num in numbers:
        if num % 2 == 0:
            numbers.remove(num)  # BAD! Modifying while iterating
    
    print(numbers)             # Output: [1, 3, 5] (might miss items)
    
    # Fix: Iterate over copy
    for num in numbers[:]:
        if num % 2 == 0:
            numbers.remove(num)


MISTAKE 8: Confusing '=' and '=='

    age = 25
    
    if age = 25:               # SyntaxError! (assignment, not comparison)
        print("Adult")
    
    # Fix: Use ==
    if age == 25:
        print("Adult")


MISTAKE 9: Not understanding scope

    def function1():
        x = 10
    
    def function2():
        print(x)               # NameError! x is local to function1
    
    function1()
    function2()
    
    # Fix: Use global or pass as parameter
    x = 10                     # Global
    
    def function2():
        print(x)               # Now works


MISTAKE 10: Shadowing built-ins

    list = [1, 2, 3]           # Shadows built-in list()
    sum = 100                  # Shadows built-in sum()
    
    # Now can't use list() or sum()
    # numbers = list("abc")    # TypeError!
    
    # Fix: Use different names
    my_list = [1, 2, 3]
    total_sum = 100

16. TIPS FOR CLEAN CODE
================================================================================

TIP 1: Use meaningful variable names

    # Bad
    x = 25
    y = "John"
    z = True
    
    # Good
    age = 25
    name = "John"
    is_student = True


TIP 2: Follow naming conventions

    # Variables and functions: snake_case
    user_name = "John"
    total_price = 99.99
    
    # Constants: UPPERCASE
    MAX_USERS = 100
    PI = 3.14159
    
    # Classes: PascalCase
    class UserAccount:
        pass


TIP 3: Initialize variables before use

    # Bad
    if condition:
        result = 10
    print(result)              # Might not be defined!
    
    # Good
    result = 0                 # Initialize first
    if condition:
        result = 10
    print(result)


TIP 4: Minimize global variables

    # Bad: Using globals everywhere
    total = 0
    count = 0
    
    def process():
        global total, count
        total += 10
        count += 1
    
    # Good: Use parameters and return values
    def process(total, count):
        total += 10
        count += 1
        return total, count
    
    total, count = 0, 0
    total, count = process(total, count)


TIP 5: Use constants for magic numbers

    # Bad
    if age >= 18:
        print("Adult")
    if score >= 18:
        print("Pass")
    
    # Good
    ADULT_AGE = 18
    PASSING_SCORE = 18
    
    if age >= ADULT_AGE:
        print("Adult")
    if score >= PASSING_SCORE:
        print("Pass")


TIP 6: Avoid single-letter variables (except loops)

    # Bad
    a = 100
    b = 200
    c = a + b
    
    # Good
    price = 100
    tax = 200
    total = price + tax
    
    # Exception: loops
    for i in range(10):        # 'i' is acceptable here
        print(i)


TIP 7: Group related variables

    # Bad
    user_name = "John"
    user_age = 25
    user_city = "NYC"
    
    # Better: Use dictionary
    user = {
        "name": "John",
        "age": 25,
        "city": "NYC"
    }


TIP 8: Use type hints (Python 3.5+)

    # Without type hints
    def greet(name):
        return "Hello, " + name
    
    # With type hints (clearer!)
    def greet(name: str) -> str:
        return "Hello, " + name
    
    age: int = 25
    price: float = 19.99
    is_active: bool = True


TIP 9: Delete unused variables

    # Bad
    temp = calculate_something()
    result = process(temp)
    # temp is no longer needed but still exists
    
    # Good
    temp = calculate_something()
    result = process(temp)
    del temp                   # Free memory


TIP 10: Use descriptive names for booleans

    # Bad
    flag = True
    check = False
    
    # Good
    is_active = True
    has_permission = False
    can_edit = True
    should_retry = False

17. SUMMARY
================================================================================

KEY POINTS TO REMEMBER:

1. VARIABLES are containers that store data in memory

2. NAMING RULES:
   - Must start with letter or underscore
   - Can contain letters, numbers, underscores
   - Case-sensitive
   - Cannot use Python keywords

3. NAMING CONVENTIONS:
   - Variables: snake_case (user_name)
   - Constants: UPPERCASE (MAX_USERS)
   - Classes: PascalCase (UserAccount)

4. DYNAMIC TYPING:
   - Variables can change type
   - Type determined automatically

5. MULTIPLE ASSIGNMENT:
   - x = y = z = 0 (same value)
   - x, y, z = 1, 2, 3 (different values)

6. SWAPPING:
   - a, b = b, a (elegant Python way)

7. CONSTANTS:
   - Use UPPERCASE names
   - Convention only (not enforced)

8. SCOPE TYPES:
   - Local: inside function
   - Enclosing: in outer function
   - Global: module level
   - Built-in: Python built-ins

9. LEGB RULE:
   - Python searches: Local → Enclosing → Global → Built-in

10. KEYWORDS:
    - 'global': modify global variable
    - 'nonlocal': modify enclosing variable

11. MEMORY REFERENCES:
    - Variables point to objects
    - Mutable objects can be modified through references
    - Use .copy() for independent copies

12. IDENTITY VS EQUALITY:
    - 'is': same object
    - '==': same value
